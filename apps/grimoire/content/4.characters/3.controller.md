# Character Controller

The `useCharacterController` composable is a facade that provides character movement with automatic animation playback. It delegates to specific implementations based on the `mode` option.

## Modes

| Mode | Implementation | Input |
|------|----------------|-------|
| `pointer` | `usePointerController` | Click-to-move |
| `keyboard` | `useKeyboardController` | WASD keys (coming soon) |

## Basic Usage

```ts
import { useCharacterController } from '~/composables/useCharacterController'

const characterRef = ref<Group>()
const { play } = useCharacterAnimations(rig)

const { moveTo, update, isMoving } = useCharacterController(
  characterRef,
  { play },
  { mode: 'pointer', speed: 3 }
)

// Move character to a point (pointer mode)
moveTo(new Vector3(5, 0, 10))

// Integrate with render loop
const { onBeforeRender } = useLoop()
onBeforeRender(({ delta }) => update(delta))
```

## The Facade

```ts
export function useCharacterController(
  character: Ref<Group | undefined>,
  animationControls: AnimationControls,
  options: CharacterControllerOptions = { speed: 3 }
)
```

### Parameters

| Param | Type | Description |
|-------|------|-------------|
| `character` | `Ref<Group \| undefined>` | Ref to the character's Three.js Group |
| `animationControls` | `AnimationControls` | Animation controls from `useCharacterAnimations` |
| `options` | `CharacterControllerOptions` | Controller configuration |

#### CharacterControllerOptions

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `mode` | `'pointer' \| 'keyboard'` | `'pointer'` | Controller mode |
| `speed` | `number` | Required | Movement speed in units/second |
| `baseAnimSpeed` | `number` | `2` | Speed the walk animation was authored for |
| `onFinishMovement` | `() => void` | `undefined` | Callback when character reaches target |

### Returns

| Property | Type | Description |
|----------|------|-------------|
| `moveTo(point)` | `(Vector3) => void` | Start moving to target point |
| `update(delta)` | `(number) => void` | Update position (call in render loop) |
| `isMoving` | `Ref<boolean>` | Whether character is currently moving |
| `target` | `Ref<Vector3 \| null>` | Current movement target (null when idle) |

## Pointer Mode

Click-to-move point-and-click character movement. This is the default mode.

### How It Works

1. **Click to move**: Call `moveTo(point)` with target Vector3
2. **Auto-play walk**: Controller plays `WALKING_A` animation
3. **Auto-rotate**: Character faces movement direction
4. **Update loop**: `update(delta)` moves character each frame
5. **Reach target**: Auto-plays `IDLE_A` when within 0.1 units
6. **Callback**: Calls `onFinishMovement()` when done

### Animation Speed Scaling

The controller scales animation playback speed to match movement speed:

```ts
const animTimeScale = speed / baseAnimSpeed

// Play walk animation at scaled speed
animationControls.play(AnimationName.WALKING_A, 0.3, animTimeScale)
```

**Example**: If walk animation was created for speed `2` and you set `speed: 4`, animation plays at `2x` speed.

::alert{type="info"}
Set `baseAnimSpeed` to match the speed the animator intended. Default is `2` units/second.
::

### Using Pointer Controller Directly

You can also use `usePointerController` directly if you only need click-to-move:

```ts
import { usePointerController } from '~/composables/usePointerController'

const { moveTo, update, isMoving } = usePointerController(
  characterRef,
  { play },
  { speed: 3 }
)
```

## Render Loop Integration

All controllers require `update(delta)` to be called every frame:

```ts
const { onBeforeRender } = useLoop()
onBeforeRender(({ delta }) => update(delta))
```

The `update` function:
- Calculates distance to target
- Rotates character using `Math.atan2(direction.x, direction.z)`
- Moves character along normalized direction vector
- Stops when distance < 0.1 units

## Full Integration Example

Here's how to integrate the controller with `Character.vue`:

```vue
<script setup lang="ts">
import { useGLTF } from '@tresjs/cientos'
import type { Group } from 'three'
import { useCharacterAnimations } from '~/composables/useCharacterAnimations'
import { useCharacterController } from '~/composables/useCharacterController'

const props = defineProps<{
  entityId: string
}>()

const gameStore = useGameStore()
const entity = computed(() => gameStore.getEntity(props.entityId))
const modelPath = computed(() => entity.value?.model!)

const { nodes } = useGLTF(modelPath.value)
const rig = computed(() => nodes.value?.Rig_Medium)

// Animation system
const { play } = useCharacterAnimations(rig)

// Character reference (Three.js Group)
const characterRef = ref<Group>()

// Sync position back to store when movement ends
function syncToStore() {
  if (!characterRef.value) return
  const pos = characterRef.value.position
  const rot = characterRef.value.rotation
  gameStore.updateEntity(props.entityId, {
    position: { x: pos.x, y: pos.y, z: pos.z },
    rotation: { x: rot.x, y: rot.y, z: rot.z },
  })
}

// Controller with store sync callback
const { moveTo, update } = useCharacterController(characterRef, { play }, {
  mode: 'pointer',
  speed: 3,
  onFinishMovement: syncToStore,
})

// Integrate with render loop
const { onBeforeRender } = useLoop()
onBeforeRender(({ delta }) => update(delta))

// Expose for parent component
defineExpose({ moveTo })
</script>

<template>
  <TresGroup
    v-if="entity"
    ref="characterRef"
    :position="[entity.position.x, entity.position.y, entity.position.z]"
    :rotation="[entity.rotation?.x ?? 0, entity.rotation?.y ?? 0, entity.rotation?.z ?? 0]"
  >
    <primitive
      v-if="rig"
      :object="rig"
      cast-shadow
      receive-shadow
    />
  </TresGroup>
</template>
```

## Click-to-Move Scene Integration

Use the controller in your scene with raycasting for click-to-move:

```vue
<script setup lang="ts">
const heroRef = ref()

// Handle ground clicks
function onGroundClick(event: { point: Vector3 }) {
  heroRef.value?.moveTo(event.point)
}
</script>

<template>
  <TresCanvas>
    <Character
      ref="heroRef"
      entity-id="hero-1"
    />

    <Floor @click="onGroundClick" />
  </TresCanvas>
</template>
```

## State Synchronization

The `onFinishMovement` callback is crucial for syncing runtime state back to the store:

```ts
function syncToStore() {
  // Only sync when movement completes (not every frame)
  gameStore.updateEntity(props.entityId, {
    position: { x: pos.x, y: pos.y, z: pos.z },
    rotation: { x: rot.x, y: rot.y, z: rot.z },
  })
}
```

::alert{type="warning"}
**Don't sync every frame**: Use `onFinishMovement` to sync only when movement completes. Syncing in `update()` would trigger reactive updates 60 times/second.
::

## Advanced: Custom Speed Profiles

Different character types can have different speeds:

```ts
// Fast character (speed 5, walk anim authored for 2)
const { moveTo: moveRogue } = useCharacterController(rogueRef, { play }, {
  mode: 'pointer',
  speed: 5,
  baseAnimSpeed: 2, // Walk plays at 2.5x speed
})

// Slow character (speed 1.5, walk anim authored for 2)
const { moveTo: moveGuardian } = useCharacterController(guardianRef, { play }, {
  mode: 'pointer',
  speed: 1.5,
  baseAnimSpeed: 2, // Walk plays at 0.75x speed
})
```

## Stopping Movement

To interrupt movement programmatically, set `target.value = null`:

```ts
const { target } = useCharacterController(characterRef, { play }, { speed: 3 })

// Stop movement
target.value = null
```

The controller will auto-play `IDLE_A` when target becomes null.

## Architecture

```
composables/
├── useCharacterController.ts   # Facade - delegates based on mode
├── usePointerController.ts     # Click-to-move implementation
└── useKeyboardController.ts    # WASD implementation (TODO)
```

The facade pattern allows:
- Consistent API regardless of input method
- Easy switching between modes
- Shared return signature (`moveTo`, `update`, `isMoving`, `target`)
