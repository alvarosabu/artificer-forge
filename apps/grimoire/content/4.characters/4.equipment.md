# Equipment

The `useEquipment` composable attaches weapon GLB models to character bone slots reactively. When equipment changes in the store, weapons appear/disappear on the character's hands.

## Basic Usage

```vue
<script setup lang="ts">
import { useGLTF } from '@tresjs/cientos'

const { nodes } = useGLTF('/models/Ranger.glb')
const rig = computed(() => nodes.value?.Rig_Medium)

const equipment = computed(() => entity.value?.equipment)
useEquipment(rig, equipment)
</script>
```

Two lines in `Character.vue` — the composable handles everything else.

## How It Works

1. `useGraph` builds a flat dictionary of named nodes from the rig
2. Finds bones by slot name (`handslotr` for main hand, `handslotl` for off hand)
3. Loads the weapon GLB via `GLTFLoader` + `DRACOLoader`
4. Clones the scene and attaches it to the bone with `bone.add(scene)`
5. Watches equipment changes reactively — swaps or removes weapons as needed

## The Composable

```ts
export function useEquipment(
  rig: Ref<TresObject3D | undefined>,
  equipment: Ref<Equipment | undefined>,
)
```

### Parameters

| Param | Type | Description |
|-------|------|-------------|
| `rig` | `Ref<TresObject3D>` | The character's armature/rig |
| `equipment` | `Ref<Equipment>` | Reactive equipment state from the store |

### Equipment Interface

```ts
export interface Equipment {
  mainHand?: string  // weapon templateId
  offHand?: string   // weapon templateId
}
```

## Bone Mapping

Equipment slots map to bone names in the character rig:

| Slot | Bone Name | Description |
|------|-----------|-------------|
| `mainHand` | `handslotr` | Right hand slot |
| `offHand` | `handslotl` | Left hand slot |

::alert{type="info"}
Blender exports remove dots from bone names. If your bones are `handslot.r` in Blender, they become `handslotr` in the exported GLB.
::

## Store Actions

Equip and unequip via the game store:

```ts
const gameStore = useGameStore()

// Equip a shortsword in main hand
gameStore.equipWeapon(entityId, 'shortsword', 'mainHand')

// Unequip
gameStore.unequipWeapon(entityId, 'mainHand')
```

## Weapon Templates

Weapons are item templates with a `model` field pointing to the GLB:

```yaml
# content/entities/items/weapons/shortsword.yaml
templateId: shortsword
type: item
subtype: weapon
name: Shortsword
model: /models/items/weapons/Sword_1Handed.glb
damage:
  dice: "1d8"
  type: slashing
properties:
  - versatile
weight: 3
value: 15
```

The `model` path is what `useEquipment` loads and attaches to the bone.

## Starting Equipment

Characters can have default equipment in their template:

```yaml
# content/entities/characters/companions/ranger.yaml
templateId: ranger
type: character
name: Elara the Ranger
model: /models/Ranger.glb
equipment:
  mainHand: shortsword
```

Or equip at runtime after spawning:

```ts
const playerId = await gameStore.spawnFromTemplate('ranger', { x: 0, y: 0, z: 0 })
gameStore.equipWeapon(playerId, 'shortsword', 'mainHand')
```

## Resource Disposal

The composable handles full cleanup:

- **On unequip**: removes mesh from bone, disposes geometry, materials, and all texture maps
- **On scope dispose**: cleans up both slots automatically
- **Race conditions**: per-slot request tracking prevents stale async loads from attaching

## Why GLTFLoader Instead of useGLTF

`useGLTF` resolves the path once via `toValue()` and doesn't re-load on reactive path changes. Equipment needs dynamic loading — weapons can be swapped at any time. `GLTFLoader.loadAsync()` gives us full control over when and what to load.

::alert{type="warning"}
Weapon GLBs compressed with Draco require the `DRACOLoader`. The composable configures this automatically using Google's CDN decoder.
::
