---
title: Damage Numbers
description: Floating damage and healing numbers anchored to characters in 3D space
---

Damage numbers are floating HTML labels that appear above a character when they take damage or receive healing. They arc upward and fade out over 1.2 seconds, driven entirely by CSS. Each number is typed and color-coded, and critical hits render larger with a glow.

## How It Works

1. The experience page calls `getCharacterRef(entityId)?.showDamage(amount, type, critical)` on the character ref
2. `Character.vue` internally runs `useDamageNumbers()`, which appends a `DamageEntry` to a reactive array
3. Each entry renders a `<DamageNumber>` component via `@tresjs/cientos`'s `<Html>` — anchored at `[0, 1.5, 0]` in the character's local space
4. The `animate-damage-arc` CSS utility plays a keyframe animation; on `animationend` the component fires `@done` and the entry is removed

## DamageType

```ts
export type DamageType =
  | 'physical'
  | 'magical'
  | 'fire'
  | 'ice'
  | 'lightning'
  | 'poison'
  | 'healing'
```

Each type maps to a distinct color in `DamageNumber.vue`:

| Type | Color |
|------|-------|
| `physical` | `#ef4444` (red) |
| `magical` | `#a855f7` (purple) |
| `fire` | `#f97316` (orange) |
| `ice` | `#22d3ee` (cyan) |
| `lightning` | `#facc15` (yellow) |
| `poison` | `#a3e635` (lime) |
| `healing` | `#22c55e` (green) |

## DamageEntry Interface

```ts
export interface DamageEntry {
  id: string       // crypto.randomUUID() — used as :key and for removeNumber
  value: number    // absolute value (Math.abs applied on creation)
  type: DamageType
  critical: boolean
  drift: number    // -1 or 1, randomly assigned, controls arc direction
}
```

## useDamageNumbers()

```ts
import { useDamageNumbers } from '~/composables/useDamageNumbers'

const { numbers, showDamage, removeNumber } = useDamageNumbers()
```

### Returns

| Property | Type | Description |
|----------|------|-------------|
| `numbers` | `Ref<DamageEntry[]>` | Reactive list of active entries |
| `showDamage(value, type, critical?)` | `Function` | Append a new entry; `critical` defaults to `false` |
| `removeNumber(id)` | `Function` | Remove entry by id (called on animation end) |

`showDamage` always stores `Math.abs(value)` — pass negative values freely, the sign is discarded. The `healing` type prepends a `+` prefix in the rendered label instead.

The composable calls `onScopeDispose` to clear the array when the owning component is destroyed, preventing orphaned DOM nodes.

## Character.vue Integration

`useDamageNumbers` runs inside `Character.vue`. The returned `showDamage` function is forwarded through `defineExpose`, making it callable from parent components via a template ref:

```ts
// Character.vue (simplified)
const { numbers, showDamage, removeNumber } = useDamageNumbers()

defineExpose({
  play,
  stop,
  currentAnimName,
  actions,
  AnimationName,
  moveTo,
  onArrive,
  showDamage,   // <-- exposed to parent
})
```

In the template, entries are rendered inside the character's `<primitive>` so they inherit the 3D transform:

```vue
<DamageNumber
  v-for="entry in numbers"
  :key="entry.id"
  :entry="entry"
  @done="removeNumber(entry.id)"
/>
```

## CharacterRef Type

Experience pages access characters via the `useSceneRefs` composable, which tracks component refs by entity id. The `CharacterRef` type exposes `showDamage` alongside the animation and movement APIs:

```ts
// Calling showDamage from an experience page
const { getCharacterRef } = useSceneRefs()

getCharacterRef(entityId)?.showDamage(25, 'fire', false)
getCharacterRef(entityId)?.showDamage(50, 'lightning', true)  // critical
getCharacterRef(entityId)?.showDamage(12, 'healing')           // healing popup
```

## DamageNumber Component

`DamageNumber.vue` wraps each entry in a `<Html>` from `@tresjs/cientos`, which projects the DOM element into 3D space with perspective scaling.

```vue
<Html :position="[0, 1.5, 0]" center :distance-factor="10">
  <div
    class="font-black font-serif pointer-events-none select-none whitespace-nowrap animate-damage-arc"
    :style="{
      '--drift': entry.drift,
      color,
      fontSize,
      textShadow: `0 0 ${entry.critical ? '14px' : '8px'} ${color}88`,
    }"
    @animationend="emit('done')"
  >
    {{ label }}
  </div>
</Html>
```

The rendered label:
- Healing numbers get a `+` prefix
- Critical numbers get a `!` suffix and render at `1.5rem` (normal hits render at `1rem`)
- `textShadow` adds a glow in the type color; the radius doubles on crits

## CSS Animation

The arc effect is defined in `apps/playground/assets/css/main.css`:

```css
@keyframes damage-arc {
  0% {
    transform: translate(0, 0) scale(1.3);
    opacity: 1;
  }
  20% {
    transform: translate(calc(var(--drift) * 15px), -20px) scale(1);
    opacity: 1;
  }
  100% {
    transform: translate(calc(var(--drift) * 45px), -80px) scale(0.8);
    opacity: 0;
  }
}

@utility animate-damage-arc {
  animation: damage-arc 1.2s ease-out forwards;
}
```

The `--drift` CSS custom property (`-1` or `1`) is set inline by the component. It controls horizontal direction: numbers arc left or right at random, which prevents a wall of text when multiple hits land at once.

Timeline summary:

| Keyframe | Behavior |
|----------|----------|
| `0%` | Pops in at 1.3x scale, full opacity |
| `20%` | Settles to 1x scale, moves 15px sideways and 20px up |
| `100%` | Floats to 45px sideways and 80px up, shrinks to 0.8x, fades out |

When the animation ends, the `animationend` DOM event fires. The component emits `done`, and the parent (`Character.vue`) calls `removeNumber(entry.id)` to clean up the reactive array.

## Command Palette Integration

`useDamageNumberCommands` registers a multi-step "Damage Numbers" group in the command palette for testing damage visuals without combat logic:

```ts
const { register, unregister } = useDamageNumberCommands(getCharacterRef)

onMounted(() => register())
onUnmounted(() => unregister())
```

The function signature:

```ts
export function useDamageNumberCommands(
  getCharacterRef: (entityId: string) => CharacterRef | null,
): { register: () => void, unregister: () => void }
```

The flow is three pages deep in the palette:

1. **Top level** — "Deal Damage" or "Heal"
2. **Entity picker** — lists all `character` entities from the store
3. **Type picker** (damage path) — the six damage types, each with a Lucide icon
4. **Crit picker** — "Normal" or "Critical!" — fires `showDamage` with a random amount (5–84) and closes the palette

The heal path skips the type picker and goes directly to the crit picker using `'healing'` as the type.

## The /combat Test Page

`pages/combat/experience.vue` is the dedicated test scene. It spawns a hero and an orc scout side by side and wires up all command groups including damage numbers.

### D Keyboard Shortcut

Press `D` while on `/combat` to fire a random damage hit on the orc scout:

```ts
const DAMAGE_TYPES = ['physical', 'magical', 'fire', 'ice', 'lightning', 'poison'] as const

onKeyDown('d', () => {
  const orc = [...gameStore.entities.values()].find(e => e.templateId === 'orc_scout')
  if (!orc) return
  const type = DAMAGE_TYPES[Math.floor(Math.random() * DAMAGE_TYPES.length)]!
  const amount = Math.floor(Math.random() * 80) + 5
  const critical = Math.random() < 0.2
  getCharacterRef(orc.id)?.showDamage(amount, type, critical)
})
```

Hold `D` down to rapidly spam numbers and verify the arc animation, drift randomization, and cleanup all function correctly. A 20% critical rate means roughly one in five hits renders at larger size with an extended glow.

## Architecture Overview

```
pages/combat/experience.vue
  ├── useSceneRefs          → setCharacterRef / getCharacterRef
  ├── useDamageNumberCommands(getCharacterRef)
  │     └── command palette: Deal Damage → entity → type → crit → showDamage()
  └── onKeyDown('d') → getCharacterRef(orc.id)?.showDamage(...)

Character.vue
  ├── useDamageNumbers()    → { numbers, showDamage, removeNumber }
  ├── <DamageNumber v-for="entry in numbers" @done="removeNumber(entry.id)" />
  └── defineExpose({ showDamage, ... })

DamageNumber.vue
  ├── <Html :position="[0, 1.5, 0]" :distance-factor="10">
  ├── class="animate-damage-arc"
  ├── :style="{ '--drift': entry.drift, color, fontSize }"
  └── @animationend → emit('done')

main.css
  ├── @keyframes damage-arc  (1.2s ease-out, uses --drift)
  └── @utility animate-damage-arc
```
