A debuff that applies a toxic green aura to the character model and triggers periodic flinch animations.

## Badge

::status-effect-badge{effect-id="poisoned"}
::

| Property | Value |
|----------|-------|
| Icon | `i-lucide-skull` |
| Color | `text-green-400` |
| Background | `bg-green-900` |

## Shader Overlay

The poisoned overlay uses Worley (cellular) noise to create a pulsing toxic aura:

:::div{class="grid grid-cols-2 gap-6 items-start"}

![Poisoned Overlay](/status/poisoned.gif)


| Parameter | Value |
|-----------|-------|
| Noise type | Worley cellular noise |
| Color | `#22ff44` (toxic green) |
| Flow direction | Upward (Y-axis) |
| Flow speed | `0.3` |

:::

The pattern slowly crawls up the character, creating a bubbling cellular texture that simulates poison coursing through the body.

### How It Works

`buildWorleyEmissiveNode()` generates a procedural Worley noise pattern in TSL. The noise is mapped to the model's UV coordinates with a time-based Y offset for upward flow. The result is applied to the material's `emissiveNode`.

## Animation Reaction

While poisoned, the character plays a periodic flinch loop:

| Parameter | Value |
|-----------|-------|
| Animation | `HIT_A` (flinch) |
| Interval | Every 3 seconds |
| Recovery animation | `SKELETONS_IDLE` |
| Recovery delay | 800ms |
| Crossfade time | 0.3s |

The loop starts when the effect is applied and stops cleanly on removal.

## Particles

No particles are spawned for poisoned.
