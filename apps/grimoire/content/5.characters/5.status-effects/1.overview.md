---
title: Status Effects
description: Visual status effects system with shader overlays, particles, and animations
---

Status effects apply visual and behavioral changes to characters and actors. Each effect combines UI badges, TSL shader overlays, particle systems, and animation reactions.

::alert{type="info"}
**Across the layers** — status effects span all three engine layers:

- **Rules** — `@artificer-forge/engine/core` (`statusEffects`) owns the canonical, pure-TS effect ids (`STATUS_EFFECT_IDS`, `StatusEffectId`). No Vue, no Three.
- **Runtime** — `@artificer-forge/engine/runtime` applies them: `useStatusEffects` plus the status-effect store hold runtime definitions and state, and `useStatusEffectOverlay` / `useStatusEffectParticles` / `useStatusEffectAnimations` drive the in-scene visuals.
- **Badges** — the visual `StatusEffectBadges` / `StatusEffectBadge` components are also exported from `@artificer-forge/engine/runtime` (they render in 3D-anchored HUD overlays like the nameplate).
::

## Store Actions

```ts
const gameStore = useGameStore()

// Add effect to entity
gameStore.addStatusEffect(entityId, 'poisoned')

// Remove effect from entity
gameStore.removeStatusEffect(entityId, 'burning')
```

Effects are stored in `EntityState.statusEffects` as an array of `StatusEffect`:

```ts
export type StatusEffectId = 'poisoned' | 'stunned' | 'burning' | 'blessed' | 'hasted' | 'frozen'

export interface StatusEffect {
  id: StatusEffectId
  turnsLeft?: number
}
```

Duplicate effects are prevented — `addStatusEffect` checks before adding.

## Status Definitions

The `STATUS_DEFINITIONS` map provides UI metadata for each effect:

```ts
import { STATUS_DEFINITIONS } from '@artificer-forge/engine/runtime'

const def = STATUS_DEFINITIONS.poisoned
// { label: 'Poisoned', icon: 'i-lucide-skull', color: 'text-green-400', bgColor: 'bg-green-900' }
```

## UI Components

### StatusEffectBadge

Renders a single effect as a colored icon with tooltip:

```vue
<StatusEffectBadge effect-id="poisoned" :turns-left="3" />
```

| Prop | Type | Description |
|------|------|-------------|
| `effectId` | `StatusEffectId` | Which effect to display |
| `turnsLeft` | `number?` | Optional turn counter chip |

### StatusEffectBadges

Container that renders multiple badges:

```vue
<StatusEffectBadges :status-effects="entity.statusEffects" direction="row" />
```

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `statusEffects` | `StatusEffect[]` | — | Effects to render |
| `direction` | `'row' \| 'col'` | `'col'` | Layout direction |

Used in `Nameplate` (horizontal) and `PartyPanel` (vertical).

## 3D Visual Effects

Three composables attach visual effects to character/actor rigs. They're integrated in `Character.vue` and `Actor.vue`.

### useStatusEffectOverlay

Applies TSL (Three.js Shading Language) procedural shader overlays to character models. Only one overlay renders at a time, using priority order: `burning > poisoned > frozen`.

```ts
const { updateOverlay, clearOverlay } = useStatusEffectOverlay(rigRef)
```

### useStatusEffectParticles

Spawns ember particles for the burning effect using `THREE.Points` with additive blending.

```ts
const { updateParticles, clearParticles } = useStatusEffectParticles(rigRef)
```

### useStatusEffectAnimations

Triggers animation reactions — currently only poisoned plays a periodic flinch.

```ts
const { updateAnimations, clearAnimations } = useStatusEffectAnimations(rigRef, play)
```

## Command Palette Integration

Status effects can be added/removed via the command palette using `useStatusEffectCommands`:

```ts
const gameStore = useGameStore()
const { pushPage, closePalette } = useCommandPalette()

useStatusEffectCommands({ gameStore, pushPage, closePalette })
```

This registers two top-level commands:
- **Add Status Effect** — pick target → pick effect (filters out already-applied)
- **Remove Status Effect** — pick target (only those with effects) → pick effect to remove

Registered in both `character/experience.vue` and `npc/experience.vue`.

## Integration Example

Status effects are fully wired in `Character.vue` and `Actor.vue`:

```vue
<script setup lang="ts">
import {
  useCharacterAnimations,
  useStatusEffectOverlay,
  useStatusEffectParticles,
  useStatusEffectAnimations,
} from '@artificer-forge/engine/runtime'

const { play } = useCharacterAnimations(rig)

// 3D visual effects
const { updateOverlay, clearOverlay } = useStatusEffectOverlay(rig)
const { updateParticles, clearParticles } = useStatusEffectParticles(rig)
const { updateAnimations, clearAnimations } = useStatusEffectAnimations(rig, play)

// React to status effect changes
watch(() => props.statusEffects, (effects) => {
  updateOverlay(effects)
  updateParticles(effects)
  updateAnimations(effects)
})

onUnmounted(() => {
  clearOverlay()
  clearParticles()
  clearAnimations()
})
</script>
```

## Architecture

```
Game Store
  ├─ addStatusEffect(entityId, effectId)
  ├─ removeStatusEffect(entityId, effectId)
  └─ EntityState.statusEffects[]

3D Components (Character.vue / Actor.vue)
  ├─ useStatusEffectOverlay   → TSL shader on model materials
  ├─ useStatusEffectParticles → THREE.Points ember system
  └─ useStatusEffectAnimations → Periodic flinch loops

UI Components
  ├─ Nameplate.vue → StatusEffectBadges (horizontal)
  └─ PartyPanel.vue → StatusEffectBadges (vertical)
      └─ StatusEffectBadge (per effect, with tooltip)

Command Palette
  └─ useStatusEffectCommands → Multi-step add/remove flow
```
