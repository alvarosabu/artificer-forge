---
title: Scenes
description: Data-driven level design with YAML scene definitions
---

Scenes define what entities exist in a level and where they're placed. This separates level design from entity templates.

## Architecture

```
Template (chest_common.yaml)     Scene (goblin-hideout.yaml)
         ↓                              ↓
    default values    +    position + rotation + overrides
                      ↓
              spawnFromTemplate()
                      ↓
              Runtime EntityState
```

| Layer | What | Example |
|-------|------|---------|
| **Templates** | What something IS | `chest_common.yaml` - model, default HP, loot table |
| **Scenes** | Where things ARE | `goblin-hideout.yaml` - positions, instance overrides |
| **Runtime** | Current state | Pinia - current HP, opened status |

## Scene Schema

```yaml
# content/scenes/goblin-hideout.yaml
sceneId: goblin_hideout
name: Goblin Hideout

spawnPoints:
  default: { x: 0, y: 0, z: 0 }
  hideout_front: { x: 0, y: 0, z: 8 }
  hideout_back: { x: -5, y: 0, z: 0 }

exits:
  - id: to_forest_front
    target: forest_clearing
    position: { x: 0, y: 0, z: 10 }

  - id: to_forest_back
    target: forest_clearing
    position: { x: -5, y: 0, z: -2 }

entities:
  - templateId: chest_common
    position: { x: 2, y: 0, z: 2 }
    rotation: { x: 0, y: 0.25, z: 0 }

  - templateId: chest_common
    position: { x: 5, y: 0, z: -3 }
    overrides:
      lootTable:
        - id: healing_potion
          quantity: 1
          chance: 1.0

  - templateId: goblin_scout
    position: { x: 10, y: 0, z: 5 }
    overrides:
      hp: 8
```

## Fields

### sceneId

Unique identifier used to load the scene:

```ts
await gameStore.loadScene('goblin_hideout')
```

### spawnPoints

Named positions where the party can spawn. Keys are exit IDs from connecting scenes:

```yaml
spawnPoints:
  default: { x: 0, y: 0, z: 0 }       # fallback
  hideout_front: { x: 0, y: 0, z: 8 } # from exit 'hideout_front'
```

When transitioning from another scene, pass the exit ID:

```ts
// Player used exit 'hideout_front' in forest_clearing
await gameStore.loadScene('goblin_hideout', 'hideout_front')
```

### exits

Define connections to other scenes. Each exit has:

- `id` - Unique identifier (used as spawnPoint key in target scene)
- `target` - Target sceneId
- `position` - Where the exit is located in 3D space

```yaml
exits:
  - id: to_forest_front
    target: forest_clearing
    position: { x: 0, y: 0, z: 10 }
```

::alert{type="info"}
Exit IDs should be unique and descriptive. The target scene needs a matching spawnPoint key.
::

### entities

Array of entity placements:

```yaml
entities:
  - templateId: chest_common          # required: template to spawn
    position: { x: 2, y: 0, z: 2 }    # required: world position
    rotation: { x: 0, y: 0.25, z: 0 } # optional: PI multipliers
    overrides:                         # optional: override template values
      hp: 10
      lootTable:
        - id: gold
          quantity: 50
          chance: 1.0
```

## Scene Transitions

To transition between scenes:

```ts
// 1. Player interacts with exit (e.g., door at position)
const exitId = 'to_forest_front'
const targetScene = 'forest_clearing'

// 2. Load target scene with exit ID as spawn point
const { spawnPoint } = await gameStore.loadScene(targetScene, exitId)

// 3. Position party at spawn point
gameStore.updateEntity(partyLeaderId, { position: spawnPoint })
```

## What Gets Cleared

When `loadScene` is called:

- **Cleared**: All non-party entities (enemies, interactables, world items)
- **Kept**: Party members, party inventory, world flags

```ts
// Party persists across scenes
gameStore.addToParty(heroId) // stays after loadScene

// World flags persist
gameStore.setFlag('quest_started', true) // stays after loadScene
```

## Content Config

Scenes collection in `content.config.ts`:

```ts
scenes: defineCollection({
  type: 'data',
  source: 'scenes/*.yaml',
  schema: z.object({
    sceneId: z.string(),
    name: z.string(),
    spawnPoints: z.record(z.object({
      x: z.number(),
      y: z.number(),
      z: z.number(),
    })),
    exits: z.array(z.object({
      id: z.string(),
      target: z.string(),
      position: z.object({
        x: z.number(),
        y: z.number(),
        z: z.number(),
      }),
    })).optional(),
    entities: z.array(z.object({
      templateId: z.string(),
      position: z.object({
        x: z.number(),
        y: z.number(),
        z: z.number(),
      }),
      rotation: z.object({
        x: z.number(),
        y: z.number(),
        z: z.number(),
      }).optional(),
      overrides: z.record(z.any()).optional(),
    })),
  }),
}),
```

## Scene Granularity

For WebGPU browser performance, keep scenes small:

- A house interior = scene
- A small courtyard = scene
- A dungeon room/floor = scene

Frequent transitions, smaller entity counts, better performance.
