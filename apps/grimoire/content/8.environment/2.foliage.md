---
title: Foliage
description: Procedural billboard foliage system with GPU instancing and TSL shaders
---

The `Foliage` component renders procedural vegetation as billboard sphere clusters. Each cluster is a merged set of planes distributed on a sphere surface, rendered via GPU instancing for efficient single-draw-call rendering.

## Installation

The component lives in `@artificer-forge/components`:

```ts
import { Foliage } from '@artificer-forge/components'
```

## Basic Usage

```vue
<script setup lang="ts">
import { Object3D } from 'three'
import { Foliage } from '@artificer-forge/components'

// Define placement points
const references = [
  { position: [2, 0, -3], scale: 1.0 },
  { position: [-4, 0, 1], scale: 0.8 },
  { position: [5, 0, 4], scale: 1.2 },
].map(({ position, scale }) => {
  const obj = new Object3D()
  obj.position.set(...position)
  obj.scale.setScalar(scale)
  obj.updateMatrixWorld()
  return obj
})
</script>

<template>
  <Foliage
    :references="references"
    color-a="#b4b536"
    color-b="#d8cf3b"
    :amount="80"
    :size="0.8"
  />
</template>
```

Each reference `Object3D` defines a cluster placement — its `position` sets where the cluster appears and `scale` controls the cluster size.

::alert{type="warning"}
You must call `updateMatrixWorld()` on each reference object before passing it to the component. The foliage system reads from `matrixWorld` to build instance matrices.
::

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `references` | `Object3D[]` | `[]` | Placement transforms — one cluster per reference |
| `amount` | `number` | `80` | Billboard planes per cluster |
| `size` | `number` | `0.8` | Individual plane size (world units) |
| `colorA` | `TresColor` | `'#b4b536'` | Shadow-side color (olive) |
| `colorB` | `TresColor` | `'#d8cf3b'` | Light-side color (yellow-green) |
| `seed` | `string` | `'foliage'` | RNG seed for deterministic randomization |
| `foliageTexture` | `Texture \| null` | `undefined` | Alpha texture for leaf cutout shape |
| `lightingDirection` | `Vector3` | `(1,1,0) normalized` | Direction vector for color mixing |
| `cameraForward` | `Vector3` | `(0,0,-1)` | Camera forward vector for billboard orientation |

## With Alpha Texture

Apply a leaf texture for shaped cutouts instead of plain quads:

```vue
<script setup lang="ts">
import { useTexture } from '@tresjs/cientos'

const { state: foliageTexture } = useTexture('/textures/foliage/foliage.png')
</script>

<template>
  <Foliage
    :references="references"
    :foliage-texture="foliageTexture"
    color-a="#b4b536"
    color-b="#d8cf3b"
  />
</template>
```

The texture's red channel is used for alpha testing with a 0.3 threshold. An SDF version of the texture is also available at `/textures/foliage/foliageSDF.png` (or `.ktx` for compressed).

## Reactive Updates

Colors and lighting direction are reactive — update them at runtime and the shader responds immediately:

```vue
<script setup lang="ts">
const colorA = ref('#b4b536')
const colorB = ref('#d8cf3b')
const lightDir = computed(() =>
  new Vector3(-5, -5, -5).normalize()
)
</script>

<template>
  <Foliage
    :references="references"
    :color-a="colorA"
    :color-b="colorB"
    :lighting-direction="lightDir"
  />
</template>
```

## How It Works

### Cluster Geometry

Each cluster is built by distributing `amount` billboard planes on a sphere surface:

1. **Spherical sampling** — Random phi/theta with radius biased toward the surface: `r = 1 - pow(rng(), 3)`. This clusters planes near the outer shell.
2. **Orientation** — Each plane is rotated to face outward from the sphere center, with a random Z rotation for variation.
3. **Soft normals** — Vertex normals are lerped 85% toward the sphere surface normal (intentionally not re-normalized) for softer, more organic lighting.
4. **Merge** — All planes are merged into a single `BufferGeometry`.

### GPU Instancing

One instance matrix is built per reference `Object3D`. Each instance gets a random up-vector rotation via the "look-at trick" — `dummy.up` is set to a random 2D angle, then `lookAt(cameraForward)` creates a camera-facing orientation with variation.

The result: one merged cluster geometry × N instances = single draw call.

### TSL Material

The material is a `MeshStandardNodeMaterial` built with TSL nodes:

| Node | Purpose |
|------|---------|
| `positionNode` | Applies per-instance transforms via `instance()` |
| `colorNode` | Mixes `colorA` ↔ `colorB` based on `normalWorld · lightingDirection` dot product |
| `opacityNode` | Texture red channel for alpha cutout |
| `castShadowNode` | Uses stable (non-rotated) UVs for consistent shadow silhouettes |
| `receivedShadowPositionNode` | Offsets by lighting direction for soft shadow reception |

Material properties: `DoubleSide`, `alphaTest: 0.3`, `depthWrite: true`, `transparent: false`.

### Seeded RNG

All randomization uses a deterministic seeded PRNG (`mulberry32` with string hashing). The same `seed` prop always produces identical geometry and placement. Change the seed to get different variations.

::alert{type="info"}
The RNG sequence is shared between geometry generation (~320 calls for 80 planes) and instance matrix building. Changing `amount` will shift the instance randomization since it consumes more RNG values.
::

## Factory Function

For advanced use cases, you can call `createFoliage()` directly instead of using the component:

```ts
import { createFoliage } from '@artificer-forge/components'

const { geometry, material, uniforms, count, dispose } = createFoliage({
  references,
  amount: 80,
  size: 0.8,
  colorA: '#b4b536',
  colorB: '#d8cf3b',
  seed: 'my-forest',
  foliageTexture: myTexture,
})

// Use with Three.js directly
const mesh = new InstancedMesh(geometry, material, count)
scene.add(mesh)

// Update uniforms
uniforms.colorA.value.set(new Color('#ff0000'))

// Cleanup
dispose()
```

### Return Value

| Property | Type | Description |
|----------|------|-------------|
| `geometry` | `BufferGeometry` | Merged billboard planes |
| `material` | `MeshStandardNodeMaterial` | TSL shader material |
| `uniforms.colorA` | `Uniform<Color>` | Shadow-side color uniform |
| `uniforms.colorB` | `Uniform<Color>` | Light-side color uniform |
| `uniforms.lightingDir` | `Uniform<Vector3>` | Lighting direction uniform |
| `count` | `number` | Instance count (= `references.length`) |
| `dispose` | `() => void` | Cleanup geometry and material |
