title: Companions
description: Playable party members that follow the active character and can be switched to on demand

Companions are additional playable characters that travel with the player's party. They belong to the `player` team, are rendered via `Character.vue`, and participate in the party system — following the active character when not controlled and becoming fully controllable when selected.

## Characteristics

:::div{class="grid grid-cols-2 gap-6 items-start"}

| Field | Value |
|-------|-------|
| `type` | `character` |
| `team` | `player` |
| `controllable` | `true` |
| Component | `Character.vue` |
| AI | `companion` — follows party leader |
| Party | Always added via `addToParty` |

:::

## Visual Distinctions

- **Nameplate**: Cyan text; shows `Lv. X · Race` on hover — only when **not** the active character
- **Outline on hover**: Cyan (`party` outline preset) — only when not active
- **PartyPanel card**: Portrait with blood fill HP overlay + selection glow

## YAML Template

Companions are defined like player characters but with an `ai` block declaring `behavior: companion` and a `followDistance`:

```yaml
templateId: zynrae
type: character
name: Zynrae Vexblade
class: rogue
level: 1
race: Drow

stats:
  hp: 10
  maxHp: 10
  ac: 13
  strength: 8
  dexterity: 18
  constitution: 12
  intelligence: 14
  wisdom: 12
  charisma: 14
  maxActions: 1
  maxBonusActions: 1

team: player
controllable: true

ai:
  behavior: companion
  followDistance: 2

portrait: /img/portraits/zynrae-portrait.png
model: /models/characters/zynrae.glb
animations:
  idle: Idle
  walk: Walk
  run: Run
  attack: Attack

equipment:
  mainHand: dagger
  offHand: dagger
```

## Spawning a Companion

Companions are spawned manually in the experience page — **after** the scene loads — so they are preserved across scene transitions:

```ts
const id = await gameStore.spawnFromTemplate('zynrae', {
  x: spawnPoint.x + 1.5,
  y: spawnPoint.y,
  z: spawnPoint.z,
})
gameStore.addToParty(id)
```

::alert{type="info"}
`loadScene` only clears non-party entities. Companions added to the party before or after loading are never removed by scene transitions.
::

## Follow Behavior

When a companion is not the selected (active) character, a watcher on the party leader's `moveTarget` drives them to follow at an offset:

```ts
watch(
  () => {
    const leaderId = gameStore.party.leader
    if (!leaderId || leaderId === companionId.value) return null
    return gameStore.getEntity(leaderId)?.moveTarget
  },
  (target) => {
    if (!target || !companionId.value) return
    if (gameStore.selectedEntityId === companionId.value) return
    const ref = getCharacterRef(companionId.value)
    ref?.moveTo(new Vector3(target.x + 1.5, target.y, target.z))
  },
)
```

::alert{type="warning"}
`moveTo` requires a `THREE.Vector3` instance — not a plain `{ x, y, z }` object — because it calls `.clone()` internally.
::

## Switching Control

The `PartyPanel` component renders a fixed portrait card per party member. Clicking a card calls `gameStore.selectEntity(id)`, transferring movement control to that character. Floor clicks then move the newly active character while the previous one resumes following.

```ts
// PartyPanel.vue
gameStore.selectEntity(member.id)
```

The active card is highlighted with a white glow border; all others show a dimmed gold border.
