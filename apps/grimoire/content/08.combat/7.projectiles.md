Projectiles are two cooperating pieces in `@artificer-forge/engine/runtime`: `useProjectile` spawns and animates a flying mesh from A to B along an arc, and `TrajectoryPreview` renders a glowing tube showing where a shot will go before it is fired.

```ts
import { useProjectile, TrajectoryPreview } from '@artificer-forge/engine/runtime'
```

## `useProjectile`

A plain composable (not a shared singleton). It uses TresJS `useTres` for the scene and `useLoop` for the per-frame update.

```ts
const { spawnProjectile } = useProjectile()
```

### Config

```ts
interface ProjectileConfig {
  from: Vector3
  to: Vector3
  speed: number
  arc: 'distance-based' | 'straight' | 'parabolic'
  model?: Group     // a GLTF model to fly (e.g. an arrow)
  visual?: 'orb'    // built-in glowing sphere + point light
  color?: string
}
```

### `spawnProjectile(config): Promise<void>`

Returns a promise that **resolves on arrival** â€” so callers (the ability system) can `await` it and apply damage exactly when the projectile lands.

```ts
await spawnProjectile({
  from: casterPos.clone(),
  to: targetPos,
  speed: 18,
  arc: 'distance-based',
  visual: 'orb',
  color: '#6ab0e8',
})
// ...lands here â†’ roll + apply damage
```

What it does each call:

1. Adds a `Group` at `from`. Visual is either the supplied `model` (cloned) or a built-in `orb` (sphere + `PointLight`).
2. Computes `duration = distance / speed`, and an `arcHeight` (`0` for `straight`, otherwise clamped `distance * 0.15`).
3. Per frame (`onBeforeRender`): advances an eased `t`, lerps position plus the parabolic arc `4hÂ·tÂ·(1âˆ’t)`, and for `model` projectiles orients the mesh along the tangent (with a +90Â° yaw so an arrow's `+X` axis points forward).
4. On arrival: removes the group, disposes geometry/materials, and resolves the promise.

::note
`arc: 'distance-based'` and `'parabolic'` both lob with the same arc-height curve; `'straight'` is a flat line. The eased `t` makes shots launch fast, hang at the apex, and accelerate down.
::

## `TrajectoryPreview`

A component that renders a pulsing emissive tube along the *same* arc curve `useProjectile` will fly, so the aiming preview matches the actual flight path. It renders nothing visible itself (`<slot />` only) â€” it manages a `Mesh` directly in the scene.

### Props

```ts
defineProps<{
  from: Vector3
  to: Vector3
  arc: 'distance-based' | 'straight' | 'parabolic'
  color?: string
  segments?: number   // curve resolution, default 32
}>()
```

### Behaviour

- Builds a `CatmullRomCurve3` through points sampled along the arc (identical math to `useProjectile`: `arcHeight = clamp(distance * 0.15)`, parabola `4hÂ·tÂ·(1âˆ’t)`), wrapped in a `TubeGeometry` (radius `0.02`).
- Material is emissive and semi-transparent; opacity pulses each frame via `useLoop`.
- Rebuilds on any prop change (`from` / `to` / `arc` / `color`) and disposes cleanly on unmount.

### Usage

```vue
<script setup lang="ts">
import { TrajectoryPreview } from '@artificer-forge/engine/runtime'
import { Vector3 } from 'three'

const from = new Vector3(0, 1.2, 0)
const to = new Vector3(5, 1, 8)
</script>

<template>
  <TrajectoryPreview
    :from="from"
    :to="to"
    arc="distance-based"
    color="#6ab0e8"
  />
</template>
```

::warning
Keep the preview's `arc` and `color` identical to the values you pass to `spawnProjectile`, otherwise the dotted aim line won't match where the shot actually lands.
::
