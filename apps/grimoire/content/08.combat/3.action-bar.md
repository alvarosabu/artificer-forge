The action bar is two parts: the `useActionBar` composable (runtime layer) that builds the player's ability slots and tracks the active weapon, and the `ActionBar` component (ui layer) that renders them.

```ts
import { useActionBar } from '@artificer-forge/engine/runtime'
import { ActionBar } from '@artificer-forge/engine/ui'
```

::note
`useActionBar` is created with `createSharedComposable` from `@vueuse/core` — every caller shares one reactive instance, so state set in the scene wiring is visible in the `ActionBar` widget and vice versa.
::

## Slots are derived from the leader's abilities

Unlike a hard-coded `DEFAULT_SLOTS` array, slots are **built reactively** from the party leader's `abilities`. A `watch` on `gameStore.party.leader` resolves each ability template via `gameStore.resolveAbility(id)` and maps it to an `ActionSlot`:

```ts
watch(() => gameStore.party.leader, async (leaderId) => {
  const entity = gameStore.getEntity(leaderId)
  const abilityIds = entity?.abilities ?? []
  const slots: ActionSlot[] = []
  for (let i = 0; i < abilityIds.length; i++) {
    const template = await gameStore.resolveAbility(abilityIds[i])
    if (!template) continue
    slots.push({
      id: `ability-${template.abilityId}`,
      label: template.name,
      icon: template.icon ?? 'i-heroicons-bolt',
      shortcut: String(i + 1),
      color: ABILITY_COLORS[template.damage?.type ?? ''] ?? 'primary',
      category: 'common',
      cost: template.cost,
      abilityId: template.abilityId,
    })
  }
  abilitySlots.value = slots
}, { immediate: true })
```

Slot color is keyed off the ability's damage type (`ABILITY_COLORS`: fire/bludgeoning/slashing → `error`, piercing/lightning → `warning`, force/ice → `info`).

## `ActionSlot` interface

```ts
export interface ActionSlot {
  id: string
  label: string
  icon: string
  shortcut?: string
  color?: 'error' | 'primary' | 'secondary' | 'success' | 'info' | 'warning' | 'neutral'
  disabled?: boolean
  category: 'common' | 'items' | 'passives'
  cost?: 'action' | 'bonusAction' | 'free'
  quantity?: number
  active?: boolean
  abilityId?: string
}
```

| Field | Description |
|-------|-------------|
| `id` | Unique slot id (`ability-<abilityId>`) |
| `label` / `icon` | Display name and Heroicons class shown on the button |
| `shortcut` | Hotkey label (`'1'`–`'0'`) |
| `color` | Nuxt UI color variant, derived from damage type |
| `category` | Which tab the slot lives under: `common` / `items` / `passives` |
| `cost` | Action economy: `action`, `bonusAction`, or `free` (rendered as a dot) |
| `quantity` | Consumable count badge (optional) |
| `abilityId` | The template id passed to `useAbilitySystem.activateAbility` |

## Returned API

```ts
const {
  playerEntity,        // ComputedRef<EntityState | null> — party leader
  playerClassId,       // ComputedRef<string | undefined>
  slots,               // ComputedRef<ActionSlot[]> — filtered by activeCategory
  activeCategory,      // Ref<'common' | 'items' | 'passives'>
  setCategory,         // (cat) => void
  activateSlot,        // (index: number) => void
  onSlotActivated,     // EventHookOn<ActionSlot>
  activeWeaponSlot,    // Readonly<Ref<'mainHand' | 'offHand' | 'none'>>
  setActiveWeaponSlot, // (slot: WeaponSlot) => void
  toggleWeaponSlot,    // () => void
} = useActionBar()
```

### `slots`

`ComputedRef<ActionSlot[]>` — the active category's slots, filtered from the derived ability slots. Reactive to both `activeCategory` and the leader's abilities.

### `activateSlot(index)`

Triggers the slot at `index` within the filtered `slots`. No-ops if the slot is missing or `disabled`. Fires the `onSlotActivated` event hook.

### `onSlotActivated`

A `createEventHook` from `@vueuse/core`. The scene wiring subscribes and forwards the slot's `abilityId` into the ability system:

```ts
const { onSlotActivated } = useActionBar()
const abilitySystem = useAbilitySystem()

onSlotActivated((slot) => {
  if (slot.abilityId) abilitySystem.activateAbility(slot.abilityId)
})
```

### Weapon slot

`activeWeaponSlot` is a readonly ref tracking which hand is active (`mainHand` / `offHand` / `none`). `setActiveWeaponSlot` sets it explicitly; `toggleWeaponSlot` flips main/off. The ability system drives this automatically — magic abilities hide weapons (`none`), ranged-projectile abilities switch to `offHand`.

## The `ActionBar` widget

`ActionBar` (ui layer) is the bottom-screen HUD. It composes the parts and owns the keyboard shortcuts via `@vueuse/core` `onKeyStroke` (not Nuxt UI `defineShortcuts`, so the component stays package-portable):

- **`1`–`9`, `0`** → `activateSlot(0..9)`
- **Tab** → `toggleWeaponSlot()`

| Sub-component (ui) | Renders |
|--------------------|---------|
| `ActionBarPortrait` | Leader portrait + HP, rendered live via `usePortraitRenderer` |
| `ActionBarWeapon` | Main/off-hand weapon swap |
| `ActionPointsGroup` | Action / bonus action / reaction pips (visual placeholder) |
| `ActionBarSlots` | The 8×3 ability grid + `ActionBarTabs` category tabs |
| `ActionBarResources` | The **End Turn** button (visual placeholder) |

```vue
<script setup lang="ts">
import { ActionBar } from '@artificer-forge/engine/ui'
</script>

<template>
  <ActionBar />
</template>
```

::warning
`ActionPointsGroup` and the End Turn button are presentational only — they do not yet spend or reset a per-turn action-point budget. The AP cost dot on each slot is likewise display-only.
::
