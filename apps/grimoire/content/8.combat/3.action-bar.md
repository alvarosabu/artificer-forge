title: Action Bar
description: Shared composable for hotkey-driven action slots
---

`useActionBar` is a singleton composable that manages the player's action slots — the hotkey buttons displayed at the bottom of the screen. It owns slot definitions, category filtering, keyboard shortcut bindings, and an event hook that fires when the player activates a slot.

```ts
import { useActionBar } from '~/composables/useActionBar'

const { slots, onSlotActivated, activateSlot } = useActionBar()
```

::alert{type="info"}
`useActionBar` is created with `createSharedComposable` from `@vueuse/core`. Every caller shares the same reactive instance — state changes in `Game.vue` are visible in the `ActionBar` UI component and vice versa.
::

## `ActionSlot` Interface

```ts
export interface ActionSlot {
  id: string
  label: string
  icon: string
  shortcut?: string
  color?: 'error' | 'primary' | 'secondary' | 'success' | 'info' | 'warning' | 'neutral'
  disabled?: boolean
  category: 'common' | 'items' | 'passives'
  cost?: 'action' | 'bonusAction' | 'free'
  quantity?: number
  active?: boolean
}
```

| Field | Required | Description |
|-------|:--------:|-------------|
| `id` | Yes | Unique string identifier — used by consumers to determine what to do |
| `label` | Yes | Display name shown in the action bar UI |
| `icon` | Yes | Heroicons class (e.g. `i-heroicons-bolt`) |
| `shortcut` | No | Keyboard key label shown on the button (`'1'`–`'0'`) |
| `color` | No | Nuxt UI color variant for the button |
| `disabled` | No | Prevents activation when `true` |
| `category` | Yes | Determines which tab the slot appears under |
| `cost` | No | Action economy cost: `'action'`, `'bonusAction'`, or `'free'` |
| `quantity` | No | Consumable count — e.g. potions remaining |
| `active` | No | Whether the slot is currently in an active/toggled state |

## Default Slots

`DEFAULT_SLOTS` is a module-level constant array. Currently it contains one slot:

```ts
const DEFAULT_SLOTS: ActionSlot[] = [
  {
    id: 'attack-melee',
    label: 'Attack',
    icon: 'i-heroicons-bolt',
    shortcut: '1',
    color: 'error',
    category: 'common',
    cost: 'action',
  },
]
```

To add a new ability, append to this array:

```ts
const DEFAULT_SLOTS: ActionSlot[] = [
  {
    id: 'attack-melee',
    label: 'Attack',
    icon: 'i-heroicons-bolt',
    shortcut: '1',
    color: 'error',
    category: 'common',
    cost: 'action',
  },
  {
    id: 'heal-potion',
    label: 'Healing Potion',
    icon: 'i-heroicons-beaker',
    shortcut: '2',
    color: 'success',
    category: 'items',
    cost: 'bonusAction',
    quantity: 3,
  },
]
```

## Categories

Slots are grouped into three categories. The `slots` computed property returns only slots matching the active category:

| Category | Intended use |
|----------|-------------|
| `common` | Standard combat actions (attack, dodge, disengage) |
| `items` | Consumables from the party inventory |
| `passives` | Toggleable passive abilities or stances |

```ts
const { activeCategory, setCategory } = useActionBar()

// Switch to items tab
setCategory('items')

console.log(activeCategory.value) // 'items'
```

## Keyboard Shortcuts

Keys `1` through `0` map to slot indices 0–9 within the active category. Bindings are registered via Nuxt UI's `defineShortcuts` inside the composable — they work globally as long as `useActionBar` is mounted.

```ts
defineShortcuts({
  '1': () => activateSlot(0),
  '2': () => activateSlot(1),
  // ...
  '0': () => activateSlot(9),
})
```

## Returned API

```ts
const {
  slots,           // ComputedRef<ActionSlot[]> — filtered by activeCategory
  activeCategory,  // Ref<'common' | 'items' | 'passives'>
  setCategory,     // (cat) => void
  activateSlot,    // (index: number) => void
  onSlotActivated, // EventHookOn<ActionSlot>
  playerEntity,    // ComputedRef<EntityState | null>
  playerClassId,   // ComputedRef<string | undefined>
} = useActionBar()
```

### `slots`

`ComputedRef<ActionSlot[]>` — the current category's slots, derived from `DEFAULT_SLOTS`. Reactive: updating `activeCategory` automatically changes this.

### `activateSlot(index)`

Triggers the slot at the given index within the filtered `slots` array. No-ops if the slot is `disabled` or the index is out of range.

```ts
// Programmatically press the first slot
activateSlot(0)
```

### `onSlotActivated`

An event hook created with `createEventHook` from `@vueuse/core`. Subscribe with a callback — it fires every time a non-disabled slot is activated, whether by keyboard shortcut or `activateSlot()`.

```ts
// The hook fires with the full ActionSlot object
onSlotActivated((slot: ActionSlot) => {
  console.log(slot.id)    // 'attack-melee'
  console.log(slot.cost)  // 'action'
})
```

Unlike `watch`, the event hook is imperative — it fires once per activation, not on every render. Multiple consumers can register separate callbacks.

### `playerEntity` and `playerClassId`

Convenience getters derived from `useGameStore`. `playerEntity` is the party leader's `EntityState`; `playerClassId` is their class string. Useful for UI components that conditionally show slots based on class.

```ts
const { playerClassId } = useActionBar()

// Show a ranger-only slot
const showRangerSlot = computed(() => playerClassId.value === 'ranger')
```

## Usage in Game.vue

`Game.vue` is the primary consumer of the event hook. It wires slot IDs to store actions:

```ts
const { onSlotActivated } = useActionBar()
const combatStore = useCombatStore()

onSlotActivated((slot) => {
  if (slot.id === 'attack-melee') {
    combatStore.enterTargeting()
  }
  // add cases here as more slots are introduced
})
```

This pattern keeps `useActionBar` free of any combat knowledge — it only fires what was activated. `Game.vue` decides what each slot means.

## Full Source

```ts
import { createSharedComposable, createEventHook } from '@vueuse/core'

export interface ActionSlot {
  id: string
  label: string
  icon: string
  shortcut?: string
  color?: 'error' | 'primary' | 'secondary' | 'success' | 'info' | 'warning' | 'neutral'
  disabled?: boolean
  category: 'common' | 'items' | 'passives'
  cost?: 'action' | 'bonusAction' | 'free'
  quantity?: number
  active?: boolean
}

const DEFAULT_SLOTS: ActionSlot[] = [
  {
    id: 'attack-melee',
    label: 'Attack',
    icon: 'i-heroicons-bolt',
    shortcut: '1',
    color: 'error',
    category: 'common',
    cost: 'action',
  },
]

export const useActionBar = createSharedComposable(() => {
  const gameStore = useGameStore()

  const activeCategory = ref<'common' | 'items' | 'passives'>('common')
  const slotActivated = createEventHook<ActionSlot>()

  const filteredSlots = computed<ActionSlot[]>(() =>
    DEFAULT_SLOTS.filter(s => s.category === activeCategory.value)
  )

  function setCategory(cat: 'common' | 'items' | 'passives') {
    activeCategory.value = cat
  }

  const playerEntity = computed(() => {
    if (!gameStore.party.leader) return null
    return gameStore.getEntity(gameStore.party.leader)
  })

  const playerClassId = computed(() => playerEntity.value?.class as string | undefined)

  function activateSlot(index: number) {
    const slot = filteredSlots.value[index]
    if (!slot || slot.disabled) return
    slotActivated.trigger(slot)
  }

  defineShortcuts({
    '1': () => activateSlot(0),
    '2': () => activateSlot(1),
    '3': () => activateSlot(2),
    '4': () => activateSlot(3),
    '5': () => activateSlot(4),
    '6': () => activateSlot(5),
    '7': () => activateSlot(6),
    '8': () => activateSlot(7),
    '9': () => activateSlot(8),
    '0': () => activateSlot(9),
  })

  return {
    playerEntity,
    playerClassId,
    slots: filteredSlots,
    activeCategory,
    setCategory,
    activateSlot,
    onSlotActivated: slotActivated.on,
  }
})
```

::alert{type="warning"}
`defineShortcuts` registers shortcuts globally while the composable is active. If you mount `useActionBar` in multiple components, shortcuts will only be registered once (because `createSharedComposable` returns the same instance), but make sure the instance is mounted before the player can interact with the scene.
::
