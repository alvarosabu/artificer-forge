Area-of-effect targeting is split cleanly across two layers: the **pure target math** lives in `@artificer-forge/engine/core`, and the **scene integration** (preview mesh, cursor tracking, pulsing) lives in `useAoESystem` in `@artificer-forge/engine/runtime`.

## Core: pure target math

`core/aoe` is Three-free. It operates on flat `{ x, z }` points (a Three `Vector3` is structurally compatible, so callers can pass one directly) and returns the ids of entities inside a shape:

```ts
import { entitiesInCircle, entitiesInCone, entitiesInLine } from '@artificer-forge/engine/core'

interface Vec2 { x: number, z: number }
type TargetEntities = Iterable<[string, { position: Vec2 }]>

entitiesInCircle(center: Vec2, radius: number, entities: TargetEntities): string[]
entitiesInCone(origin: Vec2, direction: Vec2, length: number, halfAngleDeg: number, entities: TargetEntities): string[]
entitiesInLine(start: Vec2, end: Vec2, halfWidth: number, entities: TargetEntities): string[]
```

| Function | Shape | Test |
|----------|-------|------|
| `entitiesInCircle` | Disc | distance from `center` ≤ `radius` (XZ plane) |
| `entitiesInCone` | Wedge | within `length` of `origin` **and** angle to `direction` ≤ `halfAngleDeg` |
| `entitiesInLine` | Band | projection onto `start→end` within segment, perpendicular distance ≤ `halfWidth` |

Because they are pure functions over an iterable, they are trivially unit-testable with no scene.

## Runtime: scene integration

`useAoESystem` is a `createSharedComposable` that owns the **preview mesh** and feeds the core functions on confirm. It does not import Three geometry into core — it builds the visual itself and only calls core math when resolving hits.

```ts
import { useAoESystem } from '@artificer-forge/engine/runtime'

const aoe = useAoESystem()
```

### Config

```ts
type AoEShape = 'circle' | 'cone' | 'line'

interface AoEConfig {
  shape: AoEShape
  radius?: number
  width?: number
  angle?: number
  color?: string
  range: number
}
```

### Returned API

```ts
{
  isActive,      // Readonly<Ref<boolean>>
  startPreview,  // (scene: Scene, casterPos: Vector3, config: AoEConfig) => void
  updatePreview, // (cursorPoint: Vector3, delta: number) => void
  rotatePreview, // (amount: number) => void
  confirm,       // () => string[]
  cancelPreview, // () => void
}
```

| Method | Role |
|--------|------|
| `startPreview` | Builds the shape mesh (circle/cone/line) and adds it to the scene at the caster's feet |
| `updatePreview` | Per-frame: moves/orients the preview to the cursor, pulses opacity, and turns it **red** when the cursor is beyond `range` |
| `rotatePreview` | Adjusts cone/line orientation (additive rotation) |
| `confirm` | Runs the matching core function and returns hit ids, then disposes the preview |
| `cancelPreview` | Removes and disposes the mesh, clears state |

### How confirm bridges to core

`confirm()` is where runtime hands off to the pure layer — it picks the core function by shape and passes the live entity map:

```ts
function confirm(): string[] {
  const config = currentConfig.value
  const entities = gameStore.entities  // Map<string, { position: { x, z } }>

  let ids: string[] = []
  if (config.shape === 'circle') {
    ids = entitiesInCircle(previewPosition.value, config.radius ?? 4, entities)
  }
  else if (config.shape === 'cone') {
    const dir = new Vector3().subVectors(previewPosition.value, casterPosition.value).normalize()
    ids = entitiesInCone(casterPosition.value, dir, config.radius ?? 6, (config.angle ?? 60) / 2, entities)
  }
  else if (config.shape === 'line') {
    ids = entitiesInLine(casterPosition.value, previewPosition.value, (config.width ?? 1) / 2, entities)
  }

  cancelPreview()
  return ids
}
```

::note
Note the angle convention: `AoEConfig.angle` is the **full** cone angle, while `entitiesInCone` takes a **half**-angle — hence `(config.angle ?? 60) / 2`. The same halving applies to line `width` → `halfWidth`.
::

## Wiring from the ability system

`useAbilitySystem` drives the whole flow for a `ground`-targeted ability: it calls `startPreview` on cast, the scene calls `updatePreview` each frame with the cursor point, and `confirmGroundTarget()` calls `confirm()` to get the hit ids it then damages.

```ts
aoeSystem.startPreview(scene.value, new Vector3(casterPos.x, 0, casterPos.z), {
  shape: template.aoe!.shape,
  radius: template.aoe!.radius,
  width: template.aoe!.width,
  angle: template.aoe!.angle,
  color: damageTypeStore.get(template.damage?.type ?? 'fire')?.color ?? '#ff4444',
  range: template.range?.normal ?? 10,
})
```
