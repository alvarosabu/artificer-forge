title: Combat Store

`useCombatStore` is a Pinia composition-API store that owns the **targeting cursor** â€” the crosshair, the hovered hostile, and the cursor's world point. It does not run attacks itself; on target confirmation it hands off to [`useAbilitySystem`](/combat/abilities). It reaches the 3D scene exclusively through `useSceneRefs`, keeping logic decoupled from Vue component internals.

```ts
import { useCombatStore } from '@artificer-forge/engine/runtime'

const combatStore = useCombatStore()
```

## State

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `isTargeting` | `Ref<boolean>` | `false` | Whether the player is currently picking a target |
| `hoveredTargetId` | `Ref<string \| null>` | `null` | Entity ID of the hostile under the cursor |
| `cursorWorldPoint` | `Ref<Vector3 \| null>` | `null` | Cursor position on the floor plane |

::alert{type="info"}
There is no `isAttacking` flag and no `MELEE_RANGE` / animation-timing constant here. Those moved to [`useAbilitySystem`](/combat/abilities). The store is now targeting-only.
::

## Public API

The store's `return` exposes exactly these members:

```ts
{
  isTargeting,        // Ref<boolean>
  hoveredTargetId,    // Ref<string | null>
  cursorWorldPoint,   // Ref<Vector3 | null>
  enterTargeting,     // () => void
  cancelTargeting,    // () => void
  hoverTarget,        // (entityId: string | null) => void
  confirmTarget,      // (entityId: string) => void
  updateCursorPoint,  // (point: Vector3) => void
}
```

### `enterTargeting()`

Enters targeting mode and adds a `cursor-crosshair` class to `document.body`. Called by `useAbilitySystem` when an ability with `lock-on` or `ground` targeting is activated.

```ts
function enterTargeting(): void
```

### `cancelTargeting()`

Clears `isTargeting`, `hoveredTargetId` and `cursorWorldPoint`, and removes the crosshair class. Fired on Escape, right-click, or once the ability system has collected enough targets.

```ts
function cancelTargeting(): void
```

### `hoverTarget(entityId)`

Sets `hoveredTargetId` when the cursor enters a hostile entity during targeting. Passing `null` clears it. No-ops when not targeting.

```ts
function hoverTarget(entityId: string | null): void
```

Only entities where `entity.team === 'hostile'` are accepted â€” party and neutral entities are ignored. Called from `Character`/`Actor` pointer events.

### `confirmTarget(entityId)`

The player's click entry point. Validates the entity is hostile, then routes into the ability system based on its current phase:

```ts
function confirmTarget(entityId: string): void
```

- If the active ability is **ground**-targeted, the click also confirms placement â†’ `abilitySystem.confirmGroundTarget()`.
- Otherwise it locks the target â†’ `abilitySystem.confirmLockOnTarget(entityId)`, and exits targeting once the ability system leaves the `selecting` phase.

```ts
function confirmTarget(entityId: string) {
  if (!isTargeting.value) return
  const entity = gameStore.getEntity(entityId)
  if (entity?.team !== 'hostile') return

  const abilitySystem = useAbilitySystem()
  if (abilitySystem.phase.value === 'selecting') {
    if (abilitySystem.activeAbility.value?.targeting === 'ground') {
      abilitySystem.confirmGroundTarget()
      return
    }
    abilitySystem.confirmLockOnTarget(entityId)
    if (abilitySystem.phase.value !== 'selecting') cancelTargeting()
  }
}
```

### `updateCursorPoint(point)`

Tracks the cursor's world-space position and rotates the **party leader** to face it. No-ops when not targeting.

```ts
function updateCursorPoint(point: Vector3): void
```

```ts
function updateCursorPoint(point: Vector3) {
  if (!isTargeting.value) return
  cursorWorldPoint.value = point
  const leaderId = gameStore.party.leader
  if (!leaderId) return
  getCharacterRef(leaderId)?.lookAt(point)
}
```

The cursor point is fed in from an invisible floor mesh in the scene:

```vue
<TresMesh @pointermove="(e) => combatStore.updateCursorPoint(e.point)">
  <TresPlaneGeometry :args="[100, 100]" />
  <TresMeshBasicMaterial :opacity="0" transparent />
</TresMesh>
```

## Cancellation listeners

The store registers global listeners (via `@vueuse/core` `useEventListener`) when `document` is available:

- **Escape** while targeting â†’ `abilitySystem.cancel()` (if not idle) + `cancelTargeting()`.
- **`contextmenu`** (right-click) while targeting â†’ `preventDefault()`, then the same cancel path.

This is why right-click both suppresses the browser menu and aborts the cast.
