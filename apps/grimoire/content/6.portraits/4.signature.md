---
title: Signature
description: How a portrait's cache key is derived from a character's appearance, and exactly when it invalidates and forces a re-bake.
---

A portrait's cache key is its **signature** ‚Äî a stable string built from everything that affects how the portrait *looks*, and nothing that doesn't. Same appearance produces the same signature, which is a cache hit; changed gear or model produces a new signature, which forces a re-bake.

```ts
import { portraitSignature } from '@artificer-forge/engine/runtime'
```

## What goes into the signature

```ts
interface PortraitAppearance {
  model?: string
  rig?: string
  equipment: Equipment
  background?: string // backdrop URL; part of the look
}

portraitSignature({ model, rig, equipment, background })
// -> "v7|/models/Ranger.glb|Rig_Medium|/items/sword.glb||/img/portraits/bgs/default.png"
```

The signature joins these parts with `|`:

| Part | Source | Why it matters |
|------|--------|----------------|
| `PORTRAIT_CACHE_VERSION` | constant (`v7`) | Pipeline version ‚Äî bump to invalidate everything |
| `model` | entity model URL | Different model = different face |
| `rig` | entity rig name | Different rig = different framing/proportions |
| `equipment.mainHand` | derived equipment | A visible weapon changes the portrait |
| `equipment.offHand` | derived equipment | A visible off-hand item changes the portrait |
| `background` | resolved backdrop | The backdrop is part of the rendered frame |

::alert{type="info"}
Only **`mainHand`** and **`offHand`** equipment feed the signature ‚Äî those are the slots that read in a head/bust framing. Worn armour or other slots do not currently re-bake the portrait.
::

## Why a signature instead of watching the entity

The game store replaces the **whole entity object** on every mutation ‚Äî including each movement tick, HP change, and status update. Watching the entity directly would re-bake the portrait constantly, on every step the character takes.

`usePortraitRenderer` instead computes the signature and keys its watcher on that primitive string. Moving the character (same model, rig, gear, backdrop) yields the same signature, so the watcher never fires and no bake runs. Only a genuine appearance change produces a new string.

## When it invalidates

A portrait re-bakes whenever the signature string changes ‚Äî that is, whenever any of:

- the character's **model** changes,
- the **rig** changes,
- the **main-hand** or **off-hand** item changes (equip, unequip, swap),
- the **backdrop** changes (authored `portraitBackground` added/changed/removed),
- the **cache version** `PORTRAIT_CACHE_VERSION` is bumped.

## The cache version

`PORTRAIT_CACHE_VERSION` (currently `'v7'`) is the first segment of every signature. Bump it whenever the render pipeline itself changes ‚Äî framing in `portraitRigPresets`, lighting in `PortraitLights`, capture resolution ‚Äî so portraits baked by older logic are invalidated and re-baked rather than served stale.

::alert{type="warning"}
The persistent cache is currently disabled (see [Bake Queue](/portraits/bake-queue)), so every session re-bakes regardless of signature. When re-enabling it, bump `PORTRAIT_CACHE_VERSION` so any `af:portraits` entries left in `localStorage` from a different pipeline are discarded.
::
