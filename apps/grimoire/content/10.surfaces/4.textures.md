---
title: Textures
description: How surface cells are packed into DataTextures — three RGBA channels feeding the pool, fire and variant-state materials.
---

The sim mutates a flat array of `SurfaceCell`s; the GPU needs textures. `core/surface/texture.ts` packs cells into three `Float32Array`s that back three `DataTexture`s, repacked every tick by `useSurface`.

Why three textures? The pool layer needs one scalar channel **per kind** (a single kind-id channel would interpolate into other kinds under bilinear filtering), which fills all four RGBA channels — so fire and variant state each get their own texture.

## packCells — pool channels

Liquid coverage, one kind per channel. Each channel carries that kind's `amount` (0 if the cell is a different kind):

| Channel | Kind |
|---------|------|
| R | water |
| G | oil |
| B | poison |
| A | blood |

Feeds `buildPoolSurfaceMaterial`.

## packFire — fire amount

Fire `amount` in R; G/B/A unused. A separate texture because the liquid RGBA is already full.

Feeds the charcoal / fire / ember / billboard meshes (`buildCharcoalSurfaceMaterial`, `buildFireSurfaceMaterial`, ember + flame-billboard systems).

## packState — variant state

The `frozen` / `electrified` flags, driving the icy and electric branches of the pool material:

| Channel | Value |
|---------|-------|
| R | `electrified` charge (0..1) |
| G | `frozen` (0 or 1) |
| B / A | unused |

`surfaceTexture.test.ts` confirms `packState` writes electrified to R and frozen to G.

## Texture setup

`useSurface` builds each `DataTexture` as `RGBAFormat` + `FloatType`, sized to the grid (`cols × rows`), with `LinearFilter` on both min and mag — the bilinear smoothing de-blocks the grid so cell boundaries read as smooth pool edges rather than hard squares. After each `step`, the three arrays are repacked and `texture.needsUpdate = true` is set:

```ts
step(grid, Math.min(delta, 0.1))
packCells(grid.cells, data)
packFire(grid.cells, fireData)
packState(grid.cells, stateData)
texture.needsUpdate = true
fireTexture.needsUpdate = true
stateTexture.needsUpdate = true
```

```ts
import { packCells, packFire, packState } from '@artificer-forge/engine/core'
```

::alert{type="info"}
The materials that consume these textures (`buildPoolSurfaceMaterial`, `buildFireSurfaceMaterial`, `buildCharcoalSurfaceMaterial`) live in `@artificer-forge/vfx`, not the engine. The engine packs the data; the VFX package turns it into shaded geometry.
::
