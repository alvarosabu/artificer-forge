title: Surface System
description: SurfaceSystem and useSurface — the runtime engine and renderer that bring the surface sim into a Tres scene.
---

The runtime side is two halves: `useSurface` (the singleton engine that owns the grid, textures and sim loop) and `SurfaceSystem.vue` (a pure renderer that consumes the engine's textures). Both come from `@artificer-forge/engine/runtime`.

## SurfaceSystem is a default system

`SurfaceSystem` already ships inside `<Game>`'s `#systems` slot, beside `CombatSystem`:

```vue
<slot name="systems">
  <CombatSystem />
  <SurfaceSystem />
</slot>
```

So **any scene under `<Game>` renders surfaces with no extra wiring** — you only add it yourself when overriding the `#systems` slot and composing a custom set. Drop it inside the Tres scene graph (it needs `useLoop`, which must run in a Tres component context):

```vue
<template>
  <SurfaceSystem :billboard-density="5" />
</template>
```

`billboardDensity` (default 5) is flame billboards per m² of field — a coverage/perf knob, not a per-pool count.

The component reads the engine's `texture`, `fireTexture`, `stateTexture`, `dimensions` and `isActive`, builds the pool/charcoal/fire/ember/billboard meshes only while a surface `isActive`, and drives the sim via `useLoop`:

```ts
const { texture, fireTexture, stateTexture, dimensions, isActive, tick } = useSurface()
const { onBeforeRender } = useLoop()
onBeforeRender(({ delta }) => tick(delta))
```

Idle scenes hold the mesh set at `null` → zero draw calls. `SurfaceSystem` is a pure renderer with no pointer handlers, so clicks fall through to a scene's own paint/move plane.

## useSurface — the engine

`useSurface()` returns the app-lifetime singleton engine. First call (from `SurfaceSystem`'s setup) builds it inside a detached `effectScope` so its hydration watchers survive route changes.

| Member | Purpose |
|--------|---------|
| `texture` / `fireTexture` / `stateTexture` | `DataTexture`s the renderer samples (pool channels, fire, variant state) |
| `dimensions` | World-space field extent `{ width, depth }` |
| `isActive` | Readonly — true whenever the grid holds any surface (hydrated or painted) |
| `sample(x, z)` | Read the cell under a world point |
| `seed(x, z, kind)` | Live paint — spreads + decays per kind |
| `event(x, z, event)` | Apply a `'lightning'` / `'cold'` event at a point (flood-fills the liquid pool) |
| `clear()` | Wipe all live grid state |
| `tick(delta)` | Advance the sim, re-pack textures, apply standing statuses |

`tick` runs the sim (`step`), repacks the three textures, and applies surface statuses to character entities each frame (`statusForCell` + the `warm` radiant-heat check). The idle guard on `isActive` means non-surface scenes pay almost nothing.

## Two ways to populate the field

**Declarative (level data)** — push `SurfaceSourceDef[]` into `useSurfaceStore`; `useSurface` watches the store and hydrates the grid. `lifetime` omitted ⇒ permanent pool. `frozen`/`electrified` pre-place the variant state.

```ts
import { useSurfaceStore } from '@artificer-forge/engine/runtime'
import type { SurfaceSourceDef } from '@artificer-forge/engine/core'

const surfaceStore = useSurfaceStore()
const surfaces: SurfaceSourceDef[] = [
  { kind: 'water', x: -4, z: -4, radius: 4 },                 // permanent pool
  { kind: 'water', x: 0, z: -5.5, radius: 3, frozen: true },  // ice sheet
  { kind: 'blood', x: 0, z: 5.5, radius: 3, electrified: true }, // charged hazard
]
surfaceStore.setSources(surfaces) // call surfaceStore.clearSurfaces() on unmount
```

**Live paint** — call `seed` / `event` directly from a pointer handler; this is runtime state (not store data), so it spreads and decays per kind:

```ts
const { seed, event } = useSurface()

function onPaint(e) {
  if (!e.point) return
  if (tool === 'paint') seed(e.point.x, e.point.z, kind)
  else event(e.point.x, e.point.z, tool) // 'lightning' | 'cold'
}
```

::warning
The engine is a **singleton** — live-painted state persists across scenes. Call `clear()` (paint scenes) or `surfaceStore.clearSurfaces()` (declarative scenes) on unmount so it doesn't leak into the next page.
::

::note
The textures are rendered by `@artificer-forge/vfx` materials (`buildPoolSurfaceMaterial`, `buildFireSurfaceMaterial`, etc.). See [Textures](/surfaces/textures) for the cell → channel mapping.
::
