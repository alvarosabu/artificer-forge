---
title: Scene Refs
description: Shared composable for cross-component character and interactable references
---

`useSceneRefs` is a globally shared composable that holds live references to rendered 3D components. It bridges the gap between the Pinia game store (which knows entity IDs and stats) and the Vue component tree (which holds the actual Three.js objects and imperative methods).

It lives in the engine's runtime layer, alongside the `Character` and `Interactable` components that register with it:

```ts
import { useSceneRefs } from '@artificer-forge/engine/runtime'
```

## The Problem It Solves

The game store tracks entity state — HP, position, animations — but it cannot directly call methods on a `Character.vue` instance to play an animation or move to a point. At the same time, combat logic, AI, and other systems need to trigger those imperative actions.

Scene refs solve this by providing a shared, singleton registry:

- `Character.vue` registers itself via `defineExpose` + `:ref` binding
- Combat stores and composables look up the ref by entity ID and call methods on it

Because `useSceneRefs` is built with `createSharedComposable` from VueUse, all callers share the exact same Maps — no prop drilling, no event bus.

## Interfaces

### CharacterRef

```ts
import type { Object3D, Vector3 } from 'three'
import type { EventHookOn } from '@vueuse/core'
import type { AnimationNameType, PlayOptions } from '@artificer-forge/engine/runtime'

export interface CharacterRef {
  play: (name: AnimationNameType, fadeTimeOrOpts?: number | PlayOptions, timeScale?: number) => void
  moveTo: (point: Vector3) => void
  onArrive: EventHookOn<Vector3>
  showDamage: (value: number, type: string, critical?: boolean) => void
  meleeAttack: () => void
  lookAt: (point: Vector3) => void
  attachToHand: (slot: 'mainHand' | 'offHand', object: Object3D) => void
  detachFromHand: (slot: 'mainHand' | 'offHand') => void
  cancelMovement: () => void
}
```

| Method | Description |
|--------|-------------|
| `play(name, fadeTimeOrOpts?, timeScale?)` | Play a named animation clip with optional cross-fade/options and speed |
| `moveTo(point)` | Start moving the character toward a world-space point |
| `onArrive` | VueUse event hook — subscribe to arrival at the target point |
| `showDamage(value, type, critical?)` | Spawn a floating damage number above the character |
| `meleeAttack()` | Trigger the melee attack sequence (animation + hit detection) |
| `lookAt(point)` | Instantly rotate the character to face a world-space point |
| `attachToHand(slot, object)` / `detachFromHand(slot)` | Parent/unparent a Three.js object to a hand bone |
| `cancelMovement()` | Stop any in-progress movement |

### InteractableRef

```ts
export interface InteractableRef {
  toggle: () => void
}
```

`toggle` opens or closes the interactable (e.g. a chest) depending on its current state.

## API

```ts
import { useSceneRefs } from '@artificer-forge/engine/runtime'

const {
  characterRefs,     // Ref<Map<string, CharacterRef>>
  interactableRefs,  // Ref<Map<string, InteractableRef>>
  setCharacterRef,
  getCharacterRef,
  setInteractableRef,
  getInteractableRef,
} = useSceneRefs()
```

| Symbol | Type | Description |
|--------|------|-------------|
| `characterRefs` | `Ref<Map<string, CharacterRef>>` | Raw reactive map, all registered characters |
| `interactableRefs` | `Ref<Map<string, InteractableRef>>` | Raw reactive map, all registered interactables |
| `setCharacterRef(entityId, ref)` | `(string, CharacterRef \| null) => void` | Register or unregister a character ref |
| `getCharacterRef(entityId)` | `(string) => CharacterRef \| null` | Look up a character ref by entity ID |
| `setInteractableRef(entityId, ref)` | `(string, InteractableRef \| null) => void` | Register or unregister an interactable ref |
| `getInteractableRef(entityId)` | `(string) => InteractableRef \| null` | Look up an interactable ref by entity ID |

Passing `null` to a setter removes the entry from the map — useful for cleanup on `onUnmounted`.

## Usage Pattern

There are two sides to using scene refs: registering and consuming.

### 1. Register — Experience Page Template

The experience page renders `Character.vue` components and wires up refs using Vue's `:ref` callback syntax:

```vue
<script setup lang="ts">
import { Character, useGameStore, useSceneRefs } from '@artificer-forge/engine/runtime'

const gameStore = useGameStore()
const { setCharacterRef } = useSceneRefs()

const characterEntities = computed(() =>
  [...gameStore.entities.values()].filter(e => e.type === 'character'),
)
</script>

<template>
  <TresCanvas>
    <TresPerspectiveCamera :position="[0, 5, 8]" :look-at="[0, 1, 0]" />
    <Character
      v-for="entity in characterEntities"
      :ref="(el: any) => setCharacterRef(entity.id, el)"
      :key="entity.id"
      :entity-id="entity.id"
    />
  </TresCanvas>
</template>
```

When Vue destroys a `Character` component, the `:ref` callback fires with `null`, which triggers the `delete` path inside `setCharacterRef` — automatic cleanup with no extra code.

### 2. Expose — Inside the Character component

The engine's `Character` component exposes the methods that conform to `CharacterRef` via `defineExpose`:

```vue
<script setup lang="ts">
// ... animation, movement, damage logic ...

defineExpose({
  play,
  moveTo,
  onArrive,
  showDamage,
  meleeAttack,
  lookAt,
  attachToHand,
  detachFromHand,
  cancelMovement,
})
</script>
```

### 3. Consume — Store or Composable

Any store or composable can retrieve the ref and call methods imperatively:

```ts
import { useSceneRefs } from '@artificer-forge/engine/runtime'

const { getCharacterRef } = useSceneRefs()

function triggerAttack(attackerId: string, targetId: string) {
  const attacker = getCharacterRef(attackerId)
  const target = getCharacterRef(targetId)

  if (!attacker || !target) return

  attacker.meleeAttack()

  // Subscribe to arrival before moving
  attacker.onArrive(() => {
    target.showDamage(12, 'physical')
    target.play('Hit_A')
  })
}
```

### 4. Interactables

The same pattern applies to interactable objects:

```vue
<!-- experience.vue -->
<Chest
  v-for="entity in chestEntities"
  :ref="(el: any) => setInteractableRef(entity.id, el)"
  :key="entity.id"
  :entity-id="entity.id"
/>
```

```ts
// From any composable or store
const { getInteractableRef } = useSceneRefs()

function openChest(entityId: string) {
  getInteractableRef(entityId)?.toggle()
}
```

::alert{type="info"}
`useSceneRefs` uses `createSharedComposable` — every call returns the same Maps. There is no need to pass refs down through props or inject/provide.
::

::alert{type="warning"}
Refs are only available after the component has mounted and the `:ref` callback has fired. Always guard with a null check: `getCharacterRef(id)?.play(...)`.
::
