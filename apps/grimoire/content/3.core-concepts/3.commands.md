---
title: Commands
description: Modular command palette registration for experience pages
---

`useCommands` is a composable that lets each experience page declare which command groups it needs. A single call registers the relevant commands when the page mounts and cleans them all up when it unmounts.

## The Problem It Solves

The command palette is a global dev tool that different experience pages populate differently. A character sandbox needs animation controls. A combat arena needs entity management and damage testing. An NPC scene needs recruit commands.

Without `useCommands`, each page would manually call `register`/`unregister` on multiple composables and wire up lifecycle hooks for each. `useCommands` collapses this into one line with a feature-flag object.

## Interface

```ts
interface CommandsOptions {
  entities?: boolean       // Entity management (spawn, select, remove)
  animations?: boolean     // Animation playback for the selected character
  statusEffects?: boolean  // Apply/remove status effects (poison, stun, etc.)
  recruit?: boolean        // Recruit NPCs into the party
  damageNumbers?: boolean  // Trigger floating damage numbers for testing
}
```

Every field is optional and defaults to `false` (disabled). Only the composables for enabled groups are instantiated.

## How It Works

Internally, `useCommands` maps each boolean to a command sub-composable:

| Option | Composable |
|--------|-----------|
| `animations` | `useAnimationCommands(selectedCharacterRef, close)` |
| `entities` | `useEntityCommands()` |
| `statusEffects` | `useStatusEffectCommands()` |
| `recruit` | `useRecruitCommands()` |
| `damageNumbers` | `useDamageNumberCommands(getCharacterRef)` |

Each sub-composable returns a `{ register, unregister }` pair. `useCommands` calls `register` in `onMounted` and `unregister` in `onUnmounted`:

```ts
export function useCommands(options: CommandsOptions) {
  const gameStore = useGameStore()
  const { getCharacterRef } = useSceneRefs()
  const { close } = useCommandPalette()

  const selectedCharacterRef = computed(() =>
    gameStore.selectedEntityId ? getCharacterRef(gameStore.selectedEntityId) : null,
  )

  const animations = options.animations
    ? useAnimationCommands(selectedCharacterRef, close)
    : null
  const entities = options.entities ? useEntityCommands() : null
  const statusEffects = options.statusEffects ? useStatusEffectCommands() : null
  const recruit = options.recruit ? useRecruitCommands() : null
  const damage = options.damageNumbers ? useDamageNumberCommands(getCharacterRef) : null

  onMounted(() => {
    animations?.register()
    entities?.register()
    statusEffects?.register()
    recruit?.register()
    damage?.register()
  })

  onUnmounted(() => {
    animations?.unregister()
    entities?.unregister()
    statusEffects?.unregister()
    recruit?.unregister()
    damage?.unregister()
  })
}
```

The `animations` group receives `selectedCharacterRef` — a computed ref that resolves to whichever `CharacterRef` is currently selected in the store. See [Scene Refs](/core-concepts/scene-refs) for how character refs are registered.

## Usage

Call `useCommands` at the top of your experience page's `<script setup>`:

```ts
useCommands({ entities: true, animations: true, statusEffects: true, recruit: true })
```

That single line replaces what would otherwise be five separate `onMounted`/`onUnmounted` pairs.

### Minimal Example — Animation Sandbox

```vue
<script setup lang="ts">
import { useSceneRefs } from '@artificer-forge/composables'

const gameStore = useGameStore()
const { setCharacterRef } = useSceneRefs()

useCommands({ animations: true })

onMounted(async () => {
  const playerId = await gameStore.spawnFromTemplate('hero', { x: 0, y: 0, z: 0 })
  gameStore.addToParty(playerId)
  gameStore.selectEntity(playerId)
})

const characterEntities = computed(() =>
  [...gameStore.entities.values()].filter(e => e.type === 'character'),
)
</script>

<template>
  <TresCanvas>
    <TresPerspectiveCamera :position="[5, 5, 5]" :near="0.1" :far="100" />
    <OrbitControls />
    <TresAmbientLight :intensity="0.8" />
    <Character
      v-for="entity in characterEntities"
      :ref="(el: any) => setCharacterRef(entity.id, el)"
      :key="entity.id"
      :entity-id="entity.id"
    />
  </TresCanvas>
</template>
```

### Full Example — Character Sandbox

The character basic experience enables all non-combat groups to support animation playback, entity spawning, status effect testing, and NPC recruitment:

```vue
<script setup lang="ts">
import { useSceneRefs } from '@artificer-forge/composables'

const gameStore = useGameStore()
const { setCharacterRef } = useSceneRefs()

useCommands({ entities: true, animations: true, statusEffects: true, recruit: true })

onMounted(async () => {
  const playerId = await gameStore.spawnFromTemplate('hero', { x: 0, y: 0, z: 0 })
  gameStore.addToParty(playerId)
  gameStore.selectEntity(playerId)
  gameStore.equipWeapon(playerId, 'shortsword', 'mainHand')

  const { spawnPoint } = await gameStore.loadScene('character_scene')
  gameStore.updateEntity(playerId, { position: spawnPoint })
})

const characterEntities = computed(() =>
  [...gameStore.entities.values()].filter(e => e.type === 'character'),
)
</script>

<template>
  <TresCanvas>
    <TresPerspectiveCamera :position="[5, 5, 5]" :near="0.1" :far="100" />
    <OrbitControls />
    <TresAmbientLight :intensity="0.8" />
    <TresDirectionalLight :position="[5, 5, 5]" :intensity="1.5" cast-shadow />
    <Character
      v-for="entity in characterEntities"
      :ref="(el: any) => setCharacterRef(entity.id, el)"
      :key="entity.id"
      :entity-id="entity.id"
    />
  </TresCanvas>
</template>
```

### Combat Example

Combat experiences typically skip `recruit` and add `damageNumbers` for hit testing:

```ts
useCommands({ entities: true, animations: true, statusEffects: true, damageNumbers: true })
```

::alert{type="info"}
`useCommands` must be called inside a component with a lifecycle context (`<script setup>` or `setup()`). It uses `onMounted` and `onUnmounted` internally.
::

::alert{type="warning"}
The `animations` group operates on `gameStore.selectedEntityId`. Make sure to call `gameStore.selectEntity(id)` after spawning or the animation commands will have no target.
::
