title: Game Store
description: Central Pinia store for all runtime game state
---

The game store is the single source of truth for runtime state: entities, party, inventory, world flags, and selection. Built with Pinia's composition API and shipped from the engine's runtime layer.

```ts
import { useGameStore } from '@artificer-forge/engine/runtime'

const gameStore = useGameStore()
```

## Overview

```ts
// @artificer-forge/engine/runtime — stores/game.ts
export const useGameStore = defineStore('game', () => {
  // World
  const currentScene = ref<string>('main')
  const worldFlags = ref<Record<string, boolean | number>>({})

  // Entities — every character, item and interactable lives here
  const entities = ref<Map<string, EntityState>>(new Map())

  // Party
  const party = reactive({
    members: [] as string[],
    leader: null as string | null,
  })

  // Selection & input
  const selectedEntityId = ref<string | null>(null)
  const inputBlocked = ref(false)
})
```

::alert{type="info"}
The engine never fetches content itself. The app wires a `ContentSource` once at init via `gameStore.configureContent(...)`; `spawnFromTemplate` / `loadScene` then resolve templates and scenes through it. See [Entity Templates](/entities/templates).
::

## State Domains

| Domain | Purpose |
|--------|---------|
| **entities** | All spawned game objects — characters, items (incl. inventory), interactables |
| **party** | Player-controlled characters (`members`) and their `leader` |
| **worldFlags** | Quest progress, unlocked areas, game state |
| **selectedEntityId** | Currently focused entity (targeting, UI highlighting) |

Inventory is **not** a separate domain — items are first-class entities in `entities`, tagged with `containerId`/`slot`. See [Inventory Overview](/inventory/overview).

## Entity State

```ts
export interface Position {
  x: number
  y: number
  z: number
}

export interface EntityState {
  id: string              // Unique instance ID
  templateId: string      // Reference to YAML template
  type: 'character' | 'item' | 'interactable'
  name: string
  position: Position
  rotation?: Position

  // Rendering
  model?: string
  rig?: string
  animations?: Record<string, unknown>

  // Character-specific
  faction?: string
  team?: 'player' | 'ally' | 'neutral' | 'hostile'
  controllable?: boolean
  recruitable?: boolean
  hp?: number
  maxHp?: number
  physicalArmor?: number
  magicalArmor?: number
  stats?: Record<string, number>
  ai?: { behavior: string; [key: string]: unknown }

  // Interactable-specific
  locked?: boolean
  opened?: boolean
  destructible?: boolean

  // Identity
  level?: number
  race?: string
  class?: string

  // Movement
  moveTarget?: Position | null

  // Status effects
  statusEffects?: StatusEffect[]

  // Item-instance fields (only when type === 'item')
  containerId?: string | null  // owning entity id, or null = in world
  slot?: EquipmentSlotKey       // when equipped, names the slot
  quantity?: number
}
```

::alert{type="info"}
This is an abridged shape. See `stores/game.ts` for the full `EntityState` (combat, dialog, loot, portrait fields).
::

## Scene Actions

### Load Scene

Loads a scene from YAML, clears world entities (keeps party), spawns scene entities:

```ts
async function loadScene(sceneId: string, spawnPointId: string = 'default') {
  // Resolved through the app-injected content source — the engine
  // stays decoupled from Nuxt Content / queryCollection.
  const scene = await requireContent().resolveScene(sceneId)

  if (!scene) {
    throw new Error(`Scene not found: ${sceneId}`)
  }

  // Clear only NON-party entities (and keep items owned by party members)
  for (const [id, entity] of entities.value) {
    if (party.members.includes(id)) continue
    if (entity.type === 'item' && entity.containerId && party.members.includes(entity.containerId)) continue
    entities.value.delete(id)
  }

  currentScene.value = sceneId

  // Spawn scene entities
  for (const entry of scene.entities) {
    await spawnFromTemplate(
      entry.templateId,
      entry.position,
      {
        rotation: entry.rotation,
        ...entry.overrides,
      },
    )
  }

  const spawnPoint = scene.spawnPoints[spawnPointId]
    ?? scene.spawnPoints.default

  return { scene, spawnPoint }
}
```

Usage:
```ts
// Load scene, position party at spawn point
const { spawnPoint } = await gameStore.loadScene('goblin_hideout', 'from_forest')
gameStore.updateEntity(partyLeaderId, { position: spawnPoint })
```

See [Scenes](/entities/scenes) for scene YAML format.

## Entity Actions

### Spawn from Template

Creates runtime instance from YAML template:

```ts
async function spawnFromTemplate(
  templateId: string,
  position: Position,
  overrides?: Partial<EntityState>,
): Promise<string> {
  // resolveTemplate is the gateway to the injected content source.
  const template = await requireContent().resolveTemplate(templateId)

  if (!template) {
    throw new Error(`Template not found: ${templateId}`)
  }

  const instanceId = `${template.templateId}_${Date.now()}`

  const entityState: EntityState = {
    id: instanceId,
    templateId: template.templateId,
    type: template.type,
    name: template.name,
    position,
    model: template.model,
    rig: template.rig,
    animations: template.animations,
    faction: template.faction,
    team: template.team ?? 'neutral',
    controllable: template.controllable,
    recruitable: template.recruitable,
    hp: template.stats?.hp ?? template.hp,
    maxHp: template.stats?.maxHp ?? template.maxHp,
    stats: template.stats,
    ai: template.ai,
    ...overrides,
  }

  spawnEntity(instanceId, entityState)
  // Starting equipment / loot tables are spawned as item entities here too.
  return instanceId
}
```

### CRUD Operations

```ts
function spawnEntity(id: string, state: EntityState) {
  entities.value.set(id, state)
}

function updateEntity(id: string, partial: Partial<EntityState>) {
  const entity = entities.value.get(id)
  if (entity) {
    entities.value.set(id, { ...entity, ...partial })
  }
}

function removeEntity(id: string) {
  entities.value.delete(id)
  // Cleanup: remove from party if present
  const partyIdx = party.members.indexOf(id)
  if (partyIdx > -1) {
    party.members.splice(partyIdx, 1)
  }
}

function getEntity(id: string) {
  return entities.value.get(id)
}
```

## Selection

Track which entity is currently focused (e.g. for targeting or UI highlighting):

```ts
const selectedEntityId = ref<string | null>(null)

function selectEntity(entityId: string | null) {
  selectedEntityId.value = entityId
}

const selectedEntity = computed(() =>
  selectedEntityId.value ? entities.value.get(selectedEntityId.value) ?? null : null,
)
```

## Party Actions

```ts
function addToParty(entityId: string) {
  if (!party.members.includes(entityId)) {
    party.members.push(entityId)
    if (!party.leader) {
      party.leader = entityId
    }
  }
}

function removeFromParty(entityId: string) {
  const idx = party.members.indexOf(entityId)
  if (idx > -1) {
    party.members.splice(idx, 1)
    if (party.leader === entityId) {
      party.leader = party.members[0] ?? null
    }
  }
}

function setPartyLeader(entityId: string) {
  if (party.members.includes(entityId)) {
    party.leader = entityId
  }
}
```

## Inventory Actions

Items are entities, so inventory operations are reparenting operations — every one routes through the single `moveItem` primitive:

```ts
gameStore.pickupItem(itemId, characterId)   // world → character bag
gameStore.dropItem(itemId, position)        // character → world
gameStore.transferItem(itemId, toCharacter) // character → character
gameStore.equipItem(itemId, 'mainHand')     // bag → equipment slot
gameStore.unequipItem(itemId)               // slot → bag

// Queries
gameStore.itemsIn(containerId)              // bag contents
gameStore.equippedAt(characterId, 'mainHand')
```

Each returns a `MoveResult` (`{ ok: true }` or `{ ok: false, reason }`). See [Inventory: Store API](/inventory/store-api) for the full primitive and validation rules.

## World Flags

Track persistent game state:

```ts
function setFlag(key: string, value: boolean | number) {
  worldFlags.value[key] = value
}

function getFlag(key: string): boolean | number | undefined {
  return worldFlags.value[key]
}

function hasFlag(key: string): boolean {
  return worldFlags.value[key] === true
}
```

Usage:
```ts
// Quest progress
gameStore.setFlag('quest_goblin_cleared', true)
gameStore.setFlag('goblin_kills', 5)

// Check state
if (gameStore.hasFlag('quest_goblin_cleared')) {
  // Show completion dialog
}
```

## Computed Getters

```ts
const partyEntities = computed(() =>
  party.members
    .map(id => entities.value.get(id))
    .filter((e): e is EntityState => !!e),
)

const partyCenter = computed((): Position => {
  const positions = partyEntities.value.map(e => e.position)
  if (!positions.length) return { x: 0, y: 0, z: 0 }
  return {
    x: positions.reduce((sum, p) => sum + p.x, 0) / positions.length,
    y: positions.reduce((sum, p) => sum + p.y, 0) / positions.length,
    z: positions.reduce((sum, p) => sum + p.z, 0) / positions.length,
  }
})

const hostileEntities = computed(() =>
  [...entities.value.values()].filter(e => e.team === 'hostile' && e.hp && e.hp > 0),
)

const actorEntities = computed(() =>
  [...entities.value.values()].filter(
    e => e.type === 'character' && !party.members.includes(e.id),
  ),
)
```

## Equipment Actions

Equipment is derived from item entities whose `containerId` is the character and whose `slot` is set. `equipItem`/`unequipItem` move an item into or out of a slot; `derivedEquipment` reads back what is currently worn:

```ts
// Equip an item already in the character's bag
gameStore.equipItem(swordId, 'mainHand')
gameStore.unequipItem(swordId)

// Read the equipped main/off-hand templateIds
const { mainHand, offHand } = gameStore.derivedEquipment(heroId)
```

See [Equipment](/characters/equipment) for how weapon models attach to character bones.

## Full Usage Example

```ts
import { useGameStore } from '@artificer-forge/engine/runtime'

const gameStore = useGameStore()

// Create party (once, persists across scenes)
const heroId = await gameStore.spawnFromTemplate('hero', { x: 0, y: 0, z: 0 })
gameStore.addToParty(heroId)
gameStore.selectEntity(heroId)

// Load scene - spawns world entities, returns spawn point
const { spawnPoint } = await gameStore.loadScene('goblin_hideout')
gameStore.updateEntity(heroId, { position: spawnPoint })

// Combat: deal typed damage (armor pool first, overflow to HP)
gameStore.applyDamage(targetId, 5, 'slashing')

// Loot — spawn a stack as an item entity in the hero's bag
await gameStore.spawnItemEntity('healing_potion', { containerId: heroId, quantity: 2 })

// Track progress
gameStore.setFlag('first_combat_complete', true)

// Transition to new scene via exit
const { spawnPoint: newSpawn } = await gameStore.loadScene('forest_clearing', 'hideout_exit')
gameStore.updateEntity(heroId, { position: newSpawn })
```
