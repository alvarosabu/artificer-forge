---
title: Actors
description: All characters that inhabit the world — heroes, NPCs, and enemies
---

Actors are characters that exist in the game world. Each is defined by a YAML template and spawned at runtime via the game store. They share the same entity pipeline but differ in team, AI behavior, and equipment.

## Actor Types

| Subtype | Team | Controlled by | Component |
|---------|------|---------------|-----------|
| Player character | `player` | Human input | `Character.vue` |
| Companion | `player` | Human input + follow AI | `Character.vue` |
| NPC | `neutral` | AI (idle/dialogue) | `Actor.vue` |
| Enemy | `hostile` | AI (aggressive) | `Actor.vue` |

Both rendering components — plus the behavior composable that drives AI animation — live in the engine's runtime layer:

```ts
import { Actor, Character, useActorBehavior } from '@artificer-forge/engine/runtime'
```

## How Actors Are Spawned

Actors defined in a scene YAML are spawned automatically when the scene loads:

```ts
const { spawnPoint } = await gameStore.loadScene('npc_scene')
```

Player characters are spawned manually in the experience page:

```ts
const playerId = await gameStore.spawnFromTemplate('hero', { x: 0, y: 0, z: 0 })
gameStore.addToParty(playerId)
```
