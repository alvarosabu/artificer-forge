title: Party HUD
description: PartyPanel and CharacterPanel — portrait, health, and armor for every party member
---

The party HUD is a vertical stack of portraits on the left edge of the screen. It's the player's at-a-glance status board and doubles as a click-target for the inventory and as a drop-zone for items.

## PartyPanel

A thin wrapper that iterates `gameStore.partyEntities` and renders one `CharacterPanel` per member. Drop it in any experience page that should show the party HUD — it positions itself fixed on the left edge.

`PartyPanel` and `CharacterPanel` ship from the engine's UI layer:

```ts
import { PartyPanel } from '@artificer-forge/engine/ui'
```

```vue
<!-- pages/combat/attack/index.vue -->
<template>
  <Game>
    <PartyPanel />
    <!-- … -->
  </Game>
</template>
```

```vue
<!-- components/PartyPanel.vue -->
<script setup lang="ts">
const gameStore = useGameStore()
const partyMembers = computed(() => gameStore.partyEntities)
</script>

<template>
  <div class="fixed left-4 top-1/2 -translate-y-1/2 flex flex-col gap-2 z-50">
    <CharacterPanel
      v-for="member in partyMembers"
      :key="member.id"
      :member="member"
    />
  </div>
</template>
```

::alert{type="info"}
`partyEntities` is a store getter — adding or removing a member via `addToParty` / `removeFromParty` updates the panel reactively, no manual refresh needed.
::

## CharacterPanel

The single portrait card. Receives a full `EntityState` member and renders:

- the character's portrait inside a button,
- a "blood fill" overlay that rises from the bottom as HP drops,
- the `hp / maxHp` text, HP bar, and (when applicable) split armor bar,
- vertical status-effect badges to the right,
- selection ring when the character is the currently-selected entity.

Clicking the portrait selects the member and opens the inventory focused on them. Drag-and-drop onto the portrait is **not** wired up — items are transferred via the inventory modal.

```vue
<CharacterPanel :member="entityState" />
```

### Blood Fill

A red gradient overlay fills the portrait from below as the character takes damage. The fill height is `(1 - hp / maxHp) * 100`, clamped to `[0, 100]`, with a 700ms ease-out transition — a hit makes the blood rise visibly.

```ts
const bloodFillPercent = computed(() => {
  const { hp, maxHp } = props.member
  if (!maxHp) return 0
  return Math.max(0, Math.min(100, (1 - (hp ?? 0) / maxHp) * 100))
})
```

### Selection & Click

Clicking the portrait selects the character **and** opens the inventory focused on them:

```ts
function handleClick() {
  gameStore.selectEntity(props.member.id)
  inventory.open(props.member.id)
}
```

The selected portrait gains a white border with a glow:

```ts
isSelected
  ? 'border-white shadow-[0_0_8px_#ffffff]'
  : 'border-gold-400/30 hover:border-gold-400/60'
```

### Health & Armor Display

Underneath the portrait the panel renders a small stat strip:

| Element | Source | Condition |
|---------|--------|-----------|
| `hp / maxHp` text | `member.hp`, `member.maxHp` | always |
| Red HP bar | `member.hp` / `member.maxHp` | always |
| Split armor bar | `physicalArmor`, `magicalArmor` and `derivedMaxArmor` | `!isDead && hasArmor` |

`hasArmor` mirrors the nameplate's check — true when either derived max is `> 0`. See [Combat &rarr; Armor](/combat/armor) for the data model.

### Status Effect Badges

Status effects render as a vertical column to the right of the portrait, only when present.

```vue
<button>…</button>
<StatusEffectBadges
  v-if="member.statusEffects?.length"
  :status-effects="member.statusEffects"
  direction="col"
/>
```

## Composition Summary

```
PartyPanel                       # fixed left edge, iterates party
└── CharacterPanel × N
    ├── button (portrait + blood fill + bars)
    └── StatusEffectBadges       # vertical, to the right
```
