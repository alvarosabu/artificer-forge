`@artificer-forge/engine` ships its Vue/TS source uncompiled and is consumed by a Nuxt 4 host app. This page covers the modules, build config, styles and peer dependencies needed to run it.

## Install

```bash
pnpm add @artificer-forge/engine
```

In this monorepo the engine is already a workspace package, so apps reference it as `"@artificer-forge/engine": "workspace:*"`.

## Nuxt Modules

The engine relies on the host app registering these Nuxt modules:

| Module | Why |
|--------|-----|
| `@tresjs/nuxt` | Declarative Three.js (`<TresCanvas>`, auto-imported Tres components) |
| `@nuxt/content` | YAML entity templates queried via `queryCollection` |
| `@nuxt/ui` | Styling base for the engine HUD (`/ui` layer) |
| `@pinia/nuxt` | Runtime game state (entities, party, inventory) |
| `@nuxt/devtools` | Optional — dev inspector |

## Configuration

Add the modules, transpile the engine, and import the engine styles in your main CSS.

::code-group

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: [
    '@tresjs/nuxt',
    '@nuxt/content',
    '@nuxt/ui',
    '@pinia/nuxt',
    '@nuxt/devtools', // optional
  ],
  css: ['~/assets/css/main.css'],
  build: {
    // Mandatory: the engine ships uncompiled .vue/.ts source.
    transpile: ['@artificer-forge/engine'],
  },
  compatibilityDate: '2025-01-01',
})
```

```css [assets/css/main.css]
@import "tailwindcss";
@import "@nuxt/ui";
/* Registers the engine's HUD/scene component utility classes (self-contained @source). */
@import "@artificer-forge/engine/styles.css";
```

::

### Why `build.transpile`?

The engine package exports raw `.vue` and `.ts` files (its `exports` map points straight at `./src/index.ts`). It is **not** pre-bundled. The host Nuxt app is responsible for compiling that source, so the engine must be added to `build.transpile` — otherwise Node tries to import `.vue` files directly and the build fails.

::warning
`build.transpile: ['@artificer-forge/engine']` is **mandatory**. Without it the engine's `.vue`/`.ts` source is never compiled and imports will throw at build/SSR time.
::

### Why the styles import?

Because the engine is compiled by the host app, the host's Tailwind needs to scan the engine's source to generate the utility classes used by the HUD and scene components. `@artificer-forge/engine/styles.css` is a self-contained Tailwind v4 source file:

```css
@source "./src/**/*.{vue,ts}";
```

The `@source` path is resolved relative to the engine package itself, so it works regardless of where your app lives. Import it once in your main CSS and the engine's classes are registered.

## Peer Dependencies

The engine declares these peers — install/version them in the host app:

| Package | Range | Notes |
|---------|-------|-------|
| `@tresjs/core` | `>=5.0.0` | |
| `@tresjs/cientos` | `>=5.0.0` | |
| `three` | `>=0.170.0` | WebGPU renderer is engine policy |
| `vue` | `>=3.4.0` | |
| `pinia` | `>=3.0.0` | Game state |
| `@vueuse/core` | `>=12.0.0` | |
| `@nuxt/ui` | `>=4.0.0` | **Optional** — only needed for the `/ui` HUD layer |

`@nuxt/ui` is an optional peer: if you don't use the engine's HUD (`@artificer-forge/engine/ui`) you can skip it.
