---
title: Visual Effects
description: Overview of the @artificer-forge/vfx package for combat and status effect visuals
---

`@artificer-forge/vfx` provides all visual feedback for combat and status systems: targeting indicators, floating damage numbers, TSL shader-based status overlays, and particle effects. Everything is designed for WebGPU via Three.js's TSL (Three.js Shading Language).

## Installation

```bash
pnpm add @artificer-forge/vfx
```

All exports are available from the package root:

```ts
import {
  TargetReticle,
  TargetIndicator,
  DamageNumber,
  useDamageNumbers,
  ghostMaterial,
  buildWorleyEmissiveNode,
  buildBurningEmissiveNode,
  buildFrozenEmissiveNode,
  STATUS_OVERLAY_EFFECTS,
  STATUS_OVERLAY_BUILDERS,
  createEmberSystem,
} from '@artificer-forge/vfx'
```

::alert{type="info"}
All shader work uses TSL (`three/tsl`) and WebGPU node materials (`three/webgpu`). Your renderer must be `WebGPURenderer` — the standard `WebGLRenderer` is not supported.
::

## Components

### TargetReticle

A flat, animated targeting circle rendered on the ground plane using a TSL SDF shader. Draws a ring with 8 inward-pointing triangles and a pulsing opacity. Intended for combat targeting — shown when the player hovers an enemy to select an attack target.

```vue
<TargetReticle
  :position="[entity.position.x, 0.01, entity.position.z]"
  color="#ff4444"
  :radius="0.8"
  :visible="isHovered"
/>
```

See [Target Reticle](/vfx/target-reticle) for the full API and shader breakdown.

### TargetIndicator

A vertical cylinder with a bottom-up gradient and sine-wave pulse. Wraps around a character to show selection state. Unlike `TargetReticle`, it's 3D rather than flat, making it visible even when the camera is near-horizontal.

```vue
<TargetIndicator
  :position="[0, 0, 0]"
  :radius="0.5"
  :height="1.0"
  color="#ffffff"
  :pulse-speed="2.0"
  :pulse-min="0.3"
  :pulse-max="1.0"
/>
```

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `position` | `[number, number, number]` | `[0, 0, 0]` | World position |
| `radius` | `number` | `0.5` | Cylinder radius |
| `height` | `number` | `1.0` | Cylinder height |
| `color` | `string` | `'#ffffff'` | Not yet wired into TSL — reserved |
| `pulseSpeed` | `number` | `2.0` | Sine frequency for the pulse |
| `pulseMin` | `number` | `0.3` | Minimum opacity during pulse |
| `pulseMax` | `number` | `1.0` | Maximum opacity during pulse |

### DamageNumber

A 2D HTML overlay that floats upward and fades out when an entity takes damage. Consumed via the `useDamageNumbers` composable.

```vue
<DamageNumber
  v-for="entry in damageNumbers.numbers.value"
  :key="entry.id"
  :value="entry.value"
  :type="entry.type"
  :critical="entry.critical"
  :drift="entry.drift"
  @done="damageNumbers.removeNumber(entry.id)"
/>
```

## Composables

### useDamageNumbers

Manages the reactive list of in-flight damage numbers. Call `showDamage` to push an entry; the `DamageNumber` component calls `@done` to remove it when the animation completes.

```ts
import { useDamageNumbers } from '@artificer-forge/vfx'

const { numbers, showDamage, removeNumber } = useDamageNumbers()

// Trigger a critical fire hit
showDamage(42, 'fire', true)

// Trigger a healing event
showDamage(15, 'healing')
```

#### DamageType

```ts
type DamageType = 'physical' | 'magical' | 'fire' | 'ice' | 'lightning' | 'poison' | 'healing'
```

#### DamageEntry

```ts
interface DamageEntry {
  id: string       // crypto.randomUUID()
  value: number    // always positive (Math.abs applied)
  type: DamageType
  critical: boolean
  drift: number    // -1 or 1 — horizontal float direction
}
```

::alert{type="info"}
`useDamageNumbers` calls `onScopeDispose` to clear the list when the owning component unmounts. Use it inside an experience component, not a global store.
::

## Materials

### ghostMaterial

Returns a `MeshPhysicalNodeMaterial` that renders an entity as a transparent Fresnel-shaded ghost. Edges glow bright cyan-blue (`#88ccff` at 12x intensity); the core is fully transparent. Drop-in replacement for any standard material.

```ts
import { ghostMaterial } from '@artificer-forge/vfx'

const mat = ghostMaterial()
// Assign to a Three.js mesh
mesh.material = mat
```

The Fresnel term uses `dot(normalView, positionViewDirection)` raised to power 1.5, shaped through `smoothstep` — equivalent to a Blender Layer Weight + RGB Curves setup.

### Status Effect Overlays

Three TSL emissive node builders for poisoned, burning, and frozen states. Each returns a color node you assign to `mesh.material.emissiveNode`.

```ts
import {
  buildWorleyEmissiveNode,
  buildBurningEmissiveNode,
  buildFrozenEmissiveNode,
  STATUS_OVERLAY_EFFECTS,
} from '@artificer-forge/vfx'

// Apply burning overlay
const cfg = STATUS_OVERLAY_EFFECTS.burning
mesh.material.emissiveNode = buildBurningEmissiveNode(cfg)
mesh.material.needsUpdate = true
```

#### STATUS_OVERLAY_EFFECTS

Pre-configured parameters for each status:

| ID | Color | Technique | Notes |
|----|-------|-----------|-------|
| `poisoned` | `#22ff44` | Worley noise flow | Upward-drifting cellular blobs |
| `burning` | `#ff5500` | FBM + lateral warp | Two-pass flame with heat color ramp |
| `frozen` | `#aaddff` | Hex cosine lattice | Crystal edge glow, height-masked upward |

#### StatusOverlayEffectConfig

```ts
interface StatusOverlayEffectConfig {
  color: string
  flowSpeed: number
  scale: number
  sharpness: number
  pulseSpeed: number
  intensity: number
}
```

#### STATUS_OVERLAY_BUILDERS

A record mapping each `StatusOverlayEffectId` to its builder function — useful when the status ID is dynamic:

```ts
import { STATUS_OVERLAY_BUILDERS, STATUS_OVERLAY_EFFECTS } from '@artificer-forge/vfx'

function applyStatusOverlay(mesh: Mesh, statusId: StatusOverlayEffectId) {
  const builder = STATUS_OVERLAY_BUILDERS[statusId]
  const cfg = STATUS_OVERLAY_EFFECTS[statusId]
  mesh.material.emissiveNode = builder(cfg)
  mesh.material.needsUpdate = true
}
```

## Particles

### createEmberSystem

Creates a CPU-driven `THREE.Points` ember particle system for characters on fire. Returns the `Points` object plus the underlying geometry buffers and per-particle metadata for frame-by-frame updates in your render loop.

```ts
import { createEmberSystem, CHAR_HEIGHT, RESET_Y } from '@artificer-forge/vfx'

const { points, geometry, positions, colors, meta } = createEmberSystem()
scene.add(points)

// In your animation loop — drift embers upward, reset when they pass RESET_Y
function tick(delta: number) {
  for (let i = 0; i < meta.length; i++) {
    const m = meta[i]
    positions[i * 3 + 1] += m.speed * delta
    if (positions[i * 3 + 1] > RESET_Y) {
      positions[i * 3 + 1] = m.initialY
    }
  }
  geometry.attributes.position.needsUpdate = true
}
```

#### Constants

| Export | Value | Description |
|--------|-------|-------------|
| `EMBER_COUNT` | `60` | Number of particles |
| `CHAR_HEIGHT` | `2.2` | Assumed character height in world units |
| `MAX_RADIUS` | `0.55` | Maximum spawn radius from character center |
| `RESET_Y` | `2.6` | Despawn height (`CHAR_HEIGHT + 0.4`) |

#### EmberMeta

Per-particle data used in the animation loop:

```ts
interface EmberMeta {
  angle: number        // Initial spawn angle (radians)
  radius: number       // Distance from center
  initialY: number     // Reset Y on loop (random 0–0.2)
  speed: number        // Upward drift speed (0.5–1.5)
  wobblePhase: number  // Phase offset for horizontal wobble
  wobbleFreq: number   // Wobble frequency (1.2–3.7)
}
```

#### heatColor / lerpRgb

Utility functions for heat-ramp coloring (deep red → orange → yellow). Used internally by `createEmberSystem` to assign per-particle vertex colors.

```ts
import { heatColor, lerpRgb } from '@artificer-forge/vfx'

const [r, g, b] = heatColor(0.75)  // orange-yellow
```
