## Shader Overlay

The burning overlay uses two-pass FBM (fractional Brownian motion) noise for realistic flame rendering:

:::div{class="grid grid-cols-2 gap-6 items-start"}

![Burning Overlay](/status/burning.gif)


| Parameter | Value |
|-----------|-------|
| Noise type | Two-pass FBM |
| Color ramp | Red → Orange → Yellow (3-stop gradient) |
| Flow direction | Upward |
| Flow speed | `1.2` |
| Intensity | `1.2` (brightest overlay) |

:::

### Flame Composition

- **Pass 1** — Low-frequency FBM warp creates lateral flame sway
- **Pass 2** — High-frequency FBM adds fine flame detail
- **Flicker** — Two overlapping sine waves at different frequencies produce an organic pulsing beat
- **Height mask** — Flames fade near the feet and reach full intensity from the knees up

### How It Works

`buildBurningEmissiveNode()` generates the flame pattern in TSL. The two noise passes are combined: the first warps the UV coordinates laterally, and the second adds high-frequency detail. The result is color-ramped through a 3-stop heat gradient and modulated by a height mask and flicker function before being applied to `emissiveNode`.

## Ember Particles

Rising embers are spawned using `THREE.Points` with `AdditiveBlending` for a glow effect:

| Parameter | Value |
|-----------|-------|
| Particle count | 60 |
| Character height | 2.2 units |
| Max radius | 0.55 units from character axis |
| Rise speed | 0.5–1.5 units/second (per particle) |
| Reset height | `y = 2.6` |

### Particle Behavior

Each ember particle has randomized properties:

| Property | Range | Description |
|----------|-------|-------------|
| `angle` | 0–2π | Cylindrical angle, re-randomized on reset |
| `radius` | 0.12–0.55 | Distance from character center |
| `speed` | 0.5–1.5 | Rise velocity |
| `wobblePhase` | 0–2π | Lateral sway phase offset |
| `wobbleFreq` | varied | Lateral sway frequency |

Particles rise continuously in cylindrical coordinates with sine-wave lateral wobble. When a particle exceeds the reset height, it re-randomizes all properties and picks a new heat color from the red → orange → yellow ramp.

The particle system attaches when `burning` is added and detaches when removed.

## Animation Reaction

As a DoT-type effect, burning triggers the periodic flinch loop:

| Parameter | Value |
|-----------|-------|
| Animation | `HIT_A` (flinch) |
| Interval | Every 3 seconds |
| Recovery animation | `SKELETONS_IDLE` |
| Recovery delay | 800ms |
| Crossfade time | 0.3s |

The loop starts when the effect is applied and stops cleanly on removal.
