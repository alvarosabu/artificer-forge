All inventory UI lives in `components/inventory/`. Components are dumb — they read from `useGameStore` queries and emit events for the container page to wire up.

## Component Tree

```
CharacterInventoryModal           # the UModal shell
├── EquipmentSlot × N             # one per slot, around the doll
├── CharacterPreview              # mini TresCanvas
│   └── CharacterPreviewModel     # GLTF + useEquipment + idle anim
├── BagGrid                       # the right-side grid
│   ├── ItemCell × items          # draggable cells, render item.icon
│   └── WeightBar                 # weight / capacity bar
ItemTooltip                       # hover tooltip with icon "popping out"
ItemContextMenu                   # right-click dropdown (singleton)
LootPopover                       # external — for chests/corpses
WorldItem                         # external — for items on the ground
PortraitDropZone                  # wraps party portraits as drop targets
```

## CharacterInventoryModal

The modal shell. Opens whenever `useInventory().isOpen` is true and renders the focused character.

```vue
<CharacterInventoryModal />
```

Layout: a 2-column grid. Left column is the equipment doll (helmet on top, side slots flanking the model, weapons on the bottom). Right column is the bag grid plus weight bar. Equipment slots are rendered conditionally — `hasSlot('helmet')` checks the character's `equipmentSlots` array, so non-humanoid creatures show only the slots they support.

## CharacterPreview & CharacterPreviewModel

A small `<TresCanvas>` showing the character with currently-equipped weapons attached to their bones — the **doll**. Equipment changes in the store reflect on the doll in real time.

```vue
<InventoryCharacterPreview :character-id="character.id" />
```

`CharacterPreviewModel` does three things:

1. Awaits the GLTF inside `<Suspense>`.
2. Resolves the rig by name (`nodes.value?.[entity.rig ?? 'Rig_Medium']`).
3. Calls `useEquipment(rig, derivedEquipment(id))` and `useCharacterAnimations(rig, rigSize)`.

The animation hook auto-plays the idle clip, so the doll breathes instead of T-posing.

::alert{type="info"}
The doll uses the same `useEquipment` composable as the in-world character (see [Equipment](/characters/equipment)) — there is no separate "preview" attach path.
::

## EquipmentSlot

One slot in the doll layout.

```vue
<InventoryEquipmentSlot
  slot-key="mainHand"
  :character-id="character.id"
  @context="onEquipmentContext"
/>
```

What it does:

- Reads the equipped item via `equippedAt(characterId, slotKey)`.
- Shows the item's `icon` (or a slot-default Lucide icon if empty).
- Acts as a drop zone — accepts dragged items, calls `moveItem` with the slot.
- Emits `context` on right-click so the parent can open the item menu.
- Visually pulses when a valid item is being dragged; flashes red for invalid types.

Slot label and default icon are looked up from local `SLOT_LABELS` and `SLOT_ICONS` maps — these are presentation-only and don't leak into the store.

## BagGrid

The 8 × 6 grid on the right side of the modal.

```vue
<InventoryBagGrid
  :character-id="character.id"
  @item-context="onItemContext"
/>
```

Behavior:

- Iterates `itemsIn(characterId)` (which skips equipped items).
- Fills the remaining cells with empty placeholders so the grid stays rectangular.
- Acts as a drop zone for **any** item not already loose in this bag — including unequipping by dropping into the same bag.
- Renders a `WeightBar` underneath.

The grid never paginates. If you add more rows than `ROWS_VISIBLE` (6), it scrolls internally.

## ItemCell

A single cell. Used in the bag, the loot popover, and anywhere else items need a 2D representation.

```vue
<InventoryItemCell
  :item="item"
  size="md"
  @click="..."
  @contextmenu="..."
/>
```

- Renders `item.icon` (or a cube fallback).
- Shows a small quantity badge in the bottom-right when `quantity > 1`.
- Is `draggable` — calls `useItemDrag().start(item)` on `dragstart`.
- Wraps in a `UTooltip` that renders `ItemTooltip` as the content.
- Accepts `item: null` to render an empty placeholder cell.

## ItemTooltip

The hover card that appears over items. Designed so the icon "pops out" of the top-right corner using absolute positioning and `drop-shadow`.

```vue
<InventoryItemTooltip :item="item" />
```

Shows name, subtype, and any of: damage dice, effect, weight, value, and a chip-style list of properties (`versatile`, `light`, etc.). Conditionally rendered — if the field isn't on the item, the row is omitted.

## WeightBar

```vue
<InventoryWeightBar :current="weight" :capacity="capacity" />
```

Three states based on the ratio:

| Ratio | Color | Meaning |
|-------|-------|---------|
| < 0.8 | gold | normal |
| 0.8 – 1.0 | warning | heavy |
| ≥ 1.0 | error | over capacity → encumbered |

The bar caps visually at 100% even though the underlying ratio can exceed 1.0.

## Putting It All Together

The entire inventory UI is a single line in your `app.vue` (or root layout):

```vue
<CharacterInventoryModal />
<InventoryItemContextMenu />
<InventoryLootPopover />
```

Everything else — opening, focusing a character, drag-and-drop, context menus, hotkeys — is wired through the shared composables described in [Interactions](/inventory/interactions).
