`@artificer-forge/vfx` is a **standalone package** — separate from `@artificer-forge/engine`. It bundles every piece of visual feedback the combat and status systems need: floating damage numbers, targeting indicators, TSL shader-based surface and status materials, and particle systems. Everything is authored against Three.js's TSL (Three.js Shading Language) for WebGPU.

## Installation

```bash
pnpm add @artificer-forge/vfx
```

It declares `three`, `vue`, `@tresjs/core`, and `@tresjs/cientos` as **peer dependencies** — install them in your app if they aren't there already:

```bash
pnpm add three vue @tresjs/core @tresjs/cientos
```

::alert{type="warning"}
All shader work uses TSL (`three/tsl`) and WebGPU node materials (`three/webgpu`). Your renderer must be a `WebGPURenderer` — the standard `WebGLRenderer` is not supported.
::

## What it provides

Everything is exported from the package root. There are no subpath entry points.

| Area | Exports | Docs |
|------|---------|------|
| **Damage numbers** | `DamageNumber`, `useDamageNumbers`, `DamageEntry`, `DamageType` | [Damage Numbers](/vfx/damage-numbers) |
| **Targeting** | `TargetReticle`, `TargetIndicator` | [Target Reticle](/vfx/target-reticle) |
| **Materials** | `ghostMaterial`, status-overlay builders, `buildFireSurfaceMaterial`, `buildPoolSurfaceMaterial`, `buildCharcoalSurfaceMaterial` | [Materials](/vfx/materials) |
| **Particles** | `createEmberSystem`, `createInstancedEmberSystem`, `createFireBillboards` | [Particles](/vfx/particles) |

```ts
import {
  // Components
  DamageNumber,
  TargetReticle,
  TargetIndicator,
  // Composables
  useDamageNumbers,
  // Materials
  ghostMaterial,
  buildWorleyEmissiveNode,
  buildBurningEmissiveNode,
  buildFrozenEmissiveNode,
  STATUS_OVERLAY_EFFECTS,
  STATUS_OVERLAY_BUILDERS,
  buildFireSurfaceMaterial,
  buildPoolSurfaceMaterial,
  buildCharcoalSurfaceMaterial,
  // Particles
  createEmberSystem,
  createInstancedEmberSystem,
  createFireBillboards,
} from '@artificer-forge/vfx'
```

## Components vs. helpers

The package mixes two integration styles:

- **Vue components** (`DamageNumber`, `TargetReticle`, `TargetIndicator`) drop straight into a TresJS template. They build and own their TSL materials internally.
- **Factory functions** (`ghostMaterial`, the `build*` materials, the particle systems) return plain Three.js objects — a `Material`, or a `{ mesh }` / `{ points, ... }` bundle — that you assign or `<primitive :object="…">` into your scene yourself. These give you control over update timing (per-frame texture uploads, render-loop drift) that a component can't.

::alert{type="info"}
Status overlays, fire/pool surfaces, and the fire particle systems are driven by **`DataTexture` field maps** that you pack and update each frame (the surface/combat systems in `@artificer-forge/engine` own that packing). The VFX package only provides the materials and meshes that read those textures.
::

Start with [Damage Numbers](/vfx/damage-numbers) for the simplest end-to-end example.
