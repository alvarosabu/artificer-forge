---
title: Damage Numbers
description: Floating combat damage and healing numbers with useDamageNumbers and the DamageNumber component
---

Damage numbers are the floating values that pop above a character when it takes a hit or is healed. The `@artificer-forge/vfx` package ships two pieces that work together:

- `useDamageNumbers()` — a composable that owns the reactive list of in-flight numbers.
- `DamageNumber` — a component that renders one number as an `Html` overlay inside the 3D scene, animates it on an upward arc, and emits `done` when the animation ends.

## The composable API

```ts
import { useDamageNumbers } from '@artificer-forge/vfx'

const { numbers, showDamage, removeNumber } = useDamageNumbers()
```

| Return | Type | Description |
|--------|------|-------------|
| `numbers` | `Ref<DamageEntry[]>` | Reactive list of active damage numbers — render one `DamageNumber` per entry |
| `showDamage` | `(value: number, type: DamageType, critical?: boolean) => void` | Push a new number. `value` is run through `Math.abs`, so sign is ignored |
| `removeNumber` | `(id: string) => void` | Remove one entry by id — call it from each `DamageNumber`'s `done` event |

```ts
// A 42-point critical fire hit
showDamage(42, 'fire', true)

// A 15-point heal (rendered as +15, green)
showDamage(15, 'healing')
```

::alert{type="info"}
`useDamageNumbers` registers `onScopeDispose` to clear the list when its owning component unmounts. Call it inside the experience/scene component that renders the numbers — not in a global Pinia store.
::

### `DamageType`

```ts
type DamageType =
  | 'physical' | 'magical' | 'fire' | 'ice'
  | 'lightning' | 'poison' | 'healing'
```

Each type maps to a fixed color inside `DamageNumber`:

| Type | Color | Type | Color |
|------|-------|------|-------|
| `physical` | `#ef4444` red | `lightning` | `#facc15` yellow |
| `magical` | `#a855f7` purple | `poison` | `#a3e635` lime |
| `fire` | `#f97316` orange | `healing` | `#22c55e` green |
| `ice` | `#22d3ee` cyan | | |

### `DamageEntry`

`showDamage` builds each entry for you — you only read these fields when rendering:

```ts
interface DamageEntry {
  id: string       // crypto.randomUUID()
  value: number    // always positive (Math.abs applied)
  type: DamageType
  critical: boolean
  drift: number    // -1 or 1 — random horizontal arc direction
}
```

## Rendering the numbers

`DamageNumber` takes a **single `entry` prop** — the whole `DamageEntry` object — and emits `done` (no payload) when its CSS animation finishes. Wire `done` to `removeNumber(entry.id)` so the list stays clean.

The numbers attach to a position in the 3D scene, so render the `DamageNumber`s as children of the target's group (the component places its `Html` at a local `[0, 1.5, 0]` offset, i.e. above the character's origin).

```vue
<script setup lang="ts">
import { DamageNumber, useDamageNumbers } from '@artificer-forge/vfx'
import { watch } from 'vue'
import { useCombatStore } from '~/stores/combat'

const { numbers, showDamage, removeNumber } = useDamageNumbers()
const combat = useCombatStore()

// Drive numbers off combat events
watch(() => combat.lastHit, (hit) => {
  if (hit) showDamage(hit.amount, hit.type, hit.critical)
})
</script>

<template>
  <!-- Place inside the target's group so [0, 1.5, 0] sits above its head -->
  <TresGroup :position="[target.x, target.y, target.z]">
    <DamageNumber
      v-for="entry in numbers"
      :key="entry.id"
      :entry="entry"
      @done="removeNumber(entry.id)"
    />
  </TresGroup>
</template>
```

::alert{type="warning"}
The old API exposed `value` / `type` / `critical` / `drift` as separate props. The current component takes the **whole `DamageEntry` via `:entry`** and emits `done` **without an id** — pass `entry.id` yourself in the handler.
::

## How the animation works

`DamageNumber` is a `@tresjs/cientos` `<Html>` overlay, not a mesh — it renders real DOM that tracks the 3D anchor point.

- **Label** — healing numbers get a `+` prefix; criticals get a `!` suffix (e.g. `+15`, `42!`).
- **Arc** — a one-shot `@keyframes damage-arc` (injected once into `document.head`) floats the number up and sideways over 2s, easing out and fading to 0 opacity. `entry.drift` (`-1` / `1`) sets the horizontal direction via a `--drift` CSS variable.
- **Distance scaling** — an `onBeforeRender` loop scales font size up with camera distance (`max(1, dist / 15)`), so numbers stay readable when the camera is pulled back. Criticals start at `5rem`, others at `3rem`.
- **Cleanup** — when the CSS animation ends, the element fires `animationend`, the component emits `done`, and your `removeNumber(entry.id)` drops it from `numbers`.
