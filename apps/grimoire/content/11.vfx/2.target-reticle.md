---
title: Target Reticle
description: TSL shader-based targeting circle for combat selection
---

`TargetReticle` renders a flat animated ring on the ground beneath a target. It is built entirely with TSL (Three.js Shading Language) node materials, making it fully compatible with `WebGPURenderer`. No textures — the geometry is generated procedurally from a `PlaneGeometry` and a custom SDF shader.

## Basic Usage

```vue
<script setup lang="ts">
import { TargetReticle } from '@artificer-forge/vfx'
import { useCombatStore } from '@artificer-forge/engine/runtime'
import { computed } from 'vue'

const combatStore = useCombatStore()

const reticlePosition = computed(() => {
  const target = combatStore.hoveredTarget
  return target
    ? [target.position.x, 0.01, target.position.z] as [number, number, number]
    : [0, 0.01, 0] as [number, number, number]
})
</script>

<template>
  <TargetReticle
    v-if="combatStore.isTargeting && combatStore.hoveredTargetId"
    :position="reticlePosition"
    color="#ff4444"
    :radius="0.8"
  />
</template>
```

::note
The Y offset `0.01` lifts the plane just above the floor to prevent Z-fighting. Use a slightly larger value (`0.02`) on uneven terrain.
::

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `position` | `[number, number, number]` | `[0, 0.01, 0]` | World position of the reticle center |
| `color` | `string` | `'#ffffff'` | Ring and triangle color (CSS hex) |
| `radius` | `number` | `0.8` | Ring radius in world units |
| `visible` | `boolean` | `true` | Show/hide without unmounting |

::warning
`radius` is read at component creation and passed to a TSL `uniform`. Changing it reactively after mount does not update the uniform or the `PlaneGeometry` size — both are constructed once. If you need a dynamic radius, unmount and remount the component.
::

## Shader Breakdown

The entire visual is computed in the fragment shader via TSL nodes. No vertex shader customization is needed — the plane is rotated -90° on the X axis (`rotation-x="-Math.PI / 2"`) so it lies flat on the ground.

### Geometry

```ts
const geometry = new PlaneGeometry(radius * 2.5, radius * 2.5)
```

The plane is sized at `radius * 2.5` to give the shader enough UV space. The ring sits at world-radius `r` without being clipped at the plane edges.

### Uniforms

```ts
const uRadius = uniform(float(radius))   // ring radius
const uColor  = uniform(color(colorHex)) // ring/triangle color
```

Both are TSL `uniform` nodes wired to `material.colorNode` and `material.opacityNode`.

### UV Remapping

UVs arrive in `[0, 1]`. They are remapped to a centered, radius-scaled space:

```ts
// [0,1] → [-1,1] centered, then scaled so the ring edge sits at world radius r
const centeredUV = uv().mul(2.0).sub(1.0).mul(uRadius.mul(1.25))
```

The `* 1.25` factor accounts for the plane being `radius * 2.5` wide — UV 1.0 maps to half that width, so we scale up to reach actual world-radius distance in shader space.

### Circle Border SDF

A signed distance field for the ring border:

```ts
const borderWidth = uRadius.mul(0.08)   // 8% of radius
const dist        = length(centeredUV)
const circleSdf   = abs(dist.sub(uRadius)).sub(borderWidth.mul(0.5))
const onCircle    = smoothstep(float(0.01), float(0.0), circleSdf)
```

`abs(dist - r)` measures distance from the ring edge in both directions. Subtracting `borderWidth / 2` shifts the SDF so that the region within `±borderWidth/2` of the ring is negative (inside). `smoothstep(0.01, 0.0, sdf)` converts that to a soft 1-inside / 0-outside mask with a 1px anti-aliased edge.

### 8 Inward Triangles

Eight isoceles triangles point inward from the ring edge, evenly spaced at 45° intervals:

```ts
const trianglesMask = Fn(() => {
  const mask = float(0).toVar()
  for (let i = 0; i < 8; i++) {
    const angle = float((i * Math.PI * 2) / 8)
    // Rotate UV into this triangle's local frame
    const rx = centeredUV.x.mul(cos(angle)).add(centeredUV.y.mul(sin(angle)))
    const ry = centeredUV.x.mul(sin(angle).negate()).add(centeredUV.y.mul(cos(angle)))

    const baseX      = uRadius.sub(borderWidth.mul(0.5))  // inner ring edge
    const tipX       = baseX.sub(arrowWidth)               // inward tip
    const halfBase   = arrowWidth.mul(0.5)

    // Linear width taper from tip (0) to base (halfBase)
    const t                = rx.sub(tipX).div(baseX.sub(tipX))
    const allowedHalfWidth = mix(float(0.0), halfBase, t)

    const inTriangle = step(tipX, rx)
      .mul(step(rx, baseX))
      .mul(step(abs(ry), allowedHalfWidth))

    mask.assign(max(mask, inTriangle))
  }
  return mask
})()
```

Each iteration rotates `centeredUV` so the triangle always points in the +X direction in local space. The triangle occupies `tipX ≤ rx ≤ baseX` along the radial axis and `|ry| ≤ allowedHalfWidth` along the tangential axis, where width tapers linearly from base to tip. The final mask is the union (max) of all 8 triangles.

### Pulse Animation

```ts
const pulse      = sin(time.mul(2.5)).mul(0.25).add(0.75)
// oscillates between 0.5 (min) and 1.0 (max) at 2.5 rad/s
const finalAlpha = combined.mul(pulse)
```

`sin` output is `[-1, 1]`. Multiplying by `0.25` and adding `0.75` maps it to `[0.5, 1.0]`, giving a visible but never fully-transparent pulse.

### Material Assembly

```ts
const material = new MeshBasicNodeMaterial({
  transparent: true,
  depthWrite: false,
  side: DoubleSide,
})

material.colorNode   = uColor
material.opacityNode = finalAlpha   // shape mask × pulse
```

`depthWrite: false` prevents the transparent plane from occluding objects behind it. `DoubleSide` ensures the ring is visible regardless of camera angle (important if the camera dips below ground level in dungeon scenes).

## Combat Integration

A typical combat experience uses `TargetReticle` alongside a `TargetIndicator` (vertical cylinder) to provide unambiguous targeting feedback:

```vue
<script setup lang="ts">
import { TargetReticle, TargetIndicator } from '@artificer-forge/vfx'
import { useCombatStore } from '@artificer-forge/engine/runtime'
import { computed } from 'vue'

const combatStore = useCombatStore()

const reticlePosition = computed<[number, number, number]>(() => {
  const t = combatStore.hoveredTarget
  return t ? [t.position.x, 0.01, t.position.z] : [0, 0.01, 0]
})

const indicatorPosition = computed<[number, number, number]>(() => {
  const t = combatStore.hoveredTarget
  return t ? [t.position.x, 0, t.position.z] : [0, 0, 0]
})
</script>

<template>
  <!-- Flat ring on the ground -->
  <TargetReticle
    v-if="combatStore.isTargeting && combatStore.hoveredTargetId"
    :position="reticlePosition"
    color="#ff4444"
    :radius="0.8"
  />

  <!-- Vertical cylinder around the character -->
  <TargetIndicator
    v-if="combatStore.isTargeting && combatStore.hoveredTargetId"
    :position="indicatorPosition"
    color="#ff4444"
    :radius="0.55"
    :height="2.2"
    :pulse-speed="3.0"
  />
</template>
```

::note
Pair `TargetReticle` with `TargetIndicator` for readability at all camera angles. The flat ring reads clearly from above; the vertical cylinder provides depth cues when the camera is near-horizontal.
::

## Full Component Source

```vue
<script setup lang="ts">
import { Color, DoubleSide, PlaneGeometry, Vector3Like } from 'three'
import {
  color, float, uniform, uv, length, abs, smoothstep,
  sin, time, cos, Fn, max, mix, step,
} from 'three/tsl'
import { MeshBasicNodeMaterial } from 'three/webgpu'
import { toRefs } from 'vue'

interface ReticleProps {
  position: [number, number, number]
  color: string
  radius: number
  visible: boolean
}

const props = withDefaults(defineProps<ReticleProps>(), {
  position: () => [0, 0.01, 0],
  color: '#ffffff',
  radius: 0.8,
  visible: true,
})

const { radius, color: colorProp } = toRefs(props)

const geometry = new PlaneGeometry(radius.value * 2.5, radius.value * 2.5)

const uRadius = uniform(float(radius.value))
const uColor  = uniform(color(colorProp.value))

const material = new MeshBasicNodeMaterial({
  transparent: true,
  depthWrite: false,
  side: DoubleSide,
})

const centeredUV  = uv().mul(2.0).sub(1.0).mul(uRadius.mul(1.25))
const borderWidth = uRadius.mul(0.08)
const arrowWidth  = borderWidth.mul(2.0)
const dist        = length(centeredUV)
const circleSdf   = abs(dist.sub(uRadius)).sub(borderWidth.mul(0.5))
const onCircle    = smoothstep(float(0.01), float(0.0), circleSdf)

const trianglesMask = Fn(() => {
  const mask  = float(0).toVar()
  const COUNT = 8
  for (let i = 0; i < COUNT; i++) {
    const angle    = float((i * Math.PI * 2) / COUNT)
    const cosA     = cos(angle)
    const sinA     = sin(angle)
    const rx       = centeredUV.x.mul(cosA).add(centeredUV.y.mul(sinA))
    const ry       = centeredUV.x.mul(sinA.negate()).add(centeredUV.y.mul(cosA))
    const baseX    = uRadius.sub(borderWidth.mul(0.5))
    const tipX     = baseX.sub(arrowWidth)
    const halfBase = arrowWidth.mul(0.5)
    const depth    = baseX.sub(tipX)
    const t        = rx.sub(tipX).div(depth)
    const allowedHalfWidth = mix(float(0.0), halfBase, t)
    const inTriangle = step(tipX, rx)
      .mul(step(rx, baseX))
      .mul(step(abs(ry), allowedHalfWidth))
    mask.assign(max(mask, inTriangle))
  }
  return mask
})()

const combined   = max(onCircle, trianglesMask)
const pulse      = sin(time.mul(2.5)).mul(0.25).add(0.75)
const finalAlpha = combined.mul(pulse)

material.colorNode   = uColor
material.opacityNode = finalAlpha
</script>

<template>
  <TresMesh
    :geometry="geometry"
    :position="position"
    :rotation-x="-Math.PI / 2"
    :material="material"
    :visible="visible"
  />
</template>
```
