`@artificer-forge/vfx` ships three particle factories for fire and ember effects. They cover two very different cost/control trade-offs:

| Factory | Backing | Animation | Use for |
|---------|---------|-----------|---------|
| `createEmberSystem` | `THREE.Points` | **CPU**, per-frame in your loop | A single character on fire |
| `createInstancedEmberSystem` | `InstancedMesh` | **GPU** (TSL vertex shader) | Sparks rising off a fire *surface* |
| `createFireBillboards` | `InstancedMesh` | **GPU** (TSL vertex shader) | Vertical flame billboards over a fire pool |

::note
The two GPU systems read a fire `DataTexture` so particles only appear above active fire cells. Keep that texture updated (re-pack + `needsUpdate`) each frame â€” the surface system in `@artificer-forge/engine` owns it.
::

## createEmberSystem

A CPU-driven `THREE.Points` cloud of 60 embers that drift upward around a burning character. It returns the `Points` object plus the raw buffers and per-particle metadata, so **you** advance the simulation in your render loop.

```ts
import { createEmberSystem, RESET_Y } from '@artificer-forge/vfx'

const { points, geometry, positions, colors, meta } = createEmberSystem()
characterGroup.add(points)   // local to the character

function tick(delta: number) {
  for (let i = 0; i < meta.length; i++) {
    const m = meta[i]
    positions[i * 3 + 1] += m.speed * delta              // rise
    if (positions[i * 3 + 1] > RESET_Y) {
      positions[i * 3 + 1] = m.initialY                  // respawn at base
    }
    // optional: horizontal wobble from m.wobblePhase / m.wobbleFreq
  }
  geometry.attributes.position.needsUpdate = true
}
```

The returned bundle: `{ points, geometry, positions, colors, meta }` â€” `positions`/`colors` are the `Float32Array`s backing the geometry attributes; `meta` is an `EmberMeta[]`.

```ts

  initialY: number     // reset Y on loop (0â€“0.2)

  wobblePhase: number  // phase offset for horizontal wobble
  wobbleFreq: number   // wobble frequency (1.2â€“3.7)

```

### Constants and color helpers

| Export | Value | Description |
|--------|-------|-------------|
| `EMBER_COUNT` | `60` | Number of particles |
| `CHAR_HEIGHT` | `2.2` | Assumed character height (world units) |
| `MAX_RADIUS` | `0.55` | Max spawn radius from center |
| `RESET_Y` | `2.6` | Despawn height (`CHAR_HEIGHT + 0.4`) |

`heatColor(heat)` returns an `[r, g, b]` on a deep-red â†’ orange â†’ yellow ramp; `lerpRgb(...)` is the linear interpolation it builds on. `createEmberSystem` uses them internally to color each ember; export them if you want matching colors elsewhere.


## createInstancedEmberSystem

A GPU spark system: an `InstancedMesh` of tiny camera-agnostic quads whose entire lifecycle (rise, drift, fade) runs in a TSL vertex shader â€” **zero per-frame CPU work**. Each instance loops on its own random offset and is suppressed wherever the fire texture reads empty.

```ts
import { createInstancedEmberSystem } from '@artificer-forge/vfx'

// width / depth = the fire field's world extents, fireTex = packed fire DataTexture
const { mesh } = createInstancedEmberSystem(WIDTH, DEPTH, fireTex, 200)
```

```vue
<template>
  <primitive :object="mesh" />
</template>
```

| Param | Default | Description |
|-------|---------|-------------|
| `width` | â€” | Fire field width (world units) |
| `depth` | â€” | Fire field depth (world units) |
| `fireTex` | â€” | Packed fire `DataTexture` (R = coverage) |
| `count` | `200` | Number of ember instances |

Returns `{ mesh }`. Embers rise ~2.5m over their life, drift on sine/cosine paths, fade in fast and out before reset, and render as a soft white-hot-core â†’ orange-rim circle (additive).

## createFireBillboards

Vertical flame billboards clustered around a fire pool's center. Like the instanced embers, the motion and flame shader run on the GPU â€” but each quad is **cylindrically camera-facing**, so flames stay upright and never go edge-on as the camera orbits.

```ts
import { createFireBillboards } from '@artificer-forge/vfx'

const { mesh } = createFireBillboards(WIDTH, DEPTH, fireTex, 60, 1.5)
mesh.position.set(firePoolX, 0, firePoolZ) // center on the pool
```

```vue
<template>
  <primitive :object="mesh" />
</template>
```

| Param | Default | Description |

| `fieldDepth` | â€” | Fire field depth (world units) |

Returns `{ mesh }`. Without `flipbookTex` it uses the same two-pass procedural flame shader as `buildFireSurfaceMaterial` (warp + body, 4-stop heat ramp, edge/height fades).
