---
title: Materials
description: TSL node materials — ghost, status overlays, and the fire/pool/charcoal surface materials
---

`@artificer-forge/vfx` exports a set of factory functions that build TSL node materials for WebGPU. They split into three families:

- **`ghostMaterial`** — a self-contained Fresnel ghost material with tunable uniforms.
- **Status overlays** — emissive node builders you graft onto an existing character material (`emissiveNode`) for poisoned / burning / frozen states.
- **Surface materials** — full materials for the ground-plane / billboard meshes that render fire, liquid pools, and charcoal beds, driven by a packed `DataTexture` field map.

::alert{type="warning"}
Every material here returns a `three/webgpu` node material (`MeshStandardNodeMaterial` / `MeshPhysicalNodeMaterial`). They only render under a `WebGPURenderer`.
::

## ghostMaterial

Renders an entity as a transparent, Fresnel-shaded ghost: edges glow, the core is see-through, and an animated fractal-noise term gives it an ethereal shimmer. Useful for spirits, stealth, death/respawn states.

Unlike the others, it takes an **options object** and returns **`{ material, uniforms }`** — the `uniforms` let you retune the look live without rebuilding the graph.

```ts
import { ghostMaterial } from '@artificer-forge/vfx'

const { material, uniforms } = ghostMaterial({
  color: '#88ccff',
  glowStrength: 12.0,
  fresnelPower: 1.5,
  noiseScale: 1.5,
  noiseSpeed: 0.3,
  noiseIntensity: 0.5,
})

mesh.material = material

// Later — adjust live (uniforms are TSL uniform nodes)
uniforms.glowStrength.value = 6.0
```

| Option | Default | Description |
|--------|---------|-------------|
| `color` | `'#88ccff'` | Edge glow / emissive color |
| `glowStrength` | `12.0` | Emissive multiplier on the Fresnel edge |
| `fresnelPower` | `1.5` | Falloff exponent — higher = thinner rim |
| `noiseScale` | `1.5` | Spatial frequency of the shimmer noise |
| `noiseSpeed` | `0.3` | Animation speed of the noise drift |
| `noiseIntensity` | `0.5` | How strongly noise modulates opacity |

::alert{type="info"}
`colorNode` is black and the visible color comes entirely from `emissiveNode`, so the ghost glows regardless of scene lighting. `transparent` + `depthWrite: false` + `DoubleSide` are set for you.
::

## Status overlays

Three builders return an **emissive color node** you assign to a character material's `emissiveNode`. They don't replace the material — they layer a glowing effect over whatever the character already renders.

| Export | Status | Technique |
|--------|--------|-----------|
| `buildWorleyEmissiveNode` | `poisoned` | Upward-flowing Worley cells (drifting blobs) |
| `buildBurningEmissiveNode` | `burning` | Two-pass FBM flame with lateral warp + heat ramp |
| `buildFrozenEmissiveNode` | `frozen` | Hex cosine lattice (crystal facets), spreads up from feet |

Each takes a `StatusOverlayEffectConfig`. Pre-tuned configs ship in `STATUS_OVERLAY_EFFECTS`:

```ts
import {
  STATUS_OVERLAY_EFFECTS,
  buildBurningEmissiveNode,
} from '@artificer-forge/vfx'

const cfg = STATUS_OVERLAY_EFFECTS.burning
mesh.material.emissiveNode = buildBurningEmissiveNode(cfg)
mesh.material.needsUpdate = true
```

```ts
interface StatusOverlayEffectConfig {
  color: string
  flowSpeed: number   // upward scroll speed (0 = static, e.g. frozen)
  scale: number       // noise / lattice spatial frequency
  sharpness: number   // edge/contrast exponent
  pulseSpeed: number  // brightness pulse frequency
  intensity: number   // overall emissive multiplier
}

type StatusOverlayEffectId = 'poisoned' | 'burning' | 'frozen'
```

### Dynamic dispatch

When the status id is only known at runtime, use `STATUS_OVERLAY_BUILDERS` — a record mapping each id to its builder:

```ts
import {
  STATUS_OVERLAY_BUILDERS,
  STATUS_OVERLAY_EFFECTS,
  type StatusOverlayEffectId,
} from '@artificer-forge/vfx'

function applyStatus(mesh: Mesh, id: StatusOverlayEffectId) {
  const builder = STATUS_OVERLAY_BUILDERS[id]
  mesh.material.emissiveNode = builder(STATUS_OVERLAY_EFFECTS[id])
  mesh.material.needsUpdate = true
}
```

::alert{type="info"}
`burning` and `frozen` are height-masked off `positionLocal.y` (flames build from the knees up; frost spreads from the feet and fades above mid-torso). That assumes the model's local origin is at its feet.
::

## Surface materials

These build full `MeshStandardNodeMaterial`s for the meshes that render environmental fire and liquids. All three read a packed `DataTexture` field map — the surface/combat systems in `@artificer-forge/engine` own packing and updating those textures each frame (`R/G/B/A` channels carry per-cell coverage and state).

### buildPoolSurfaceMaterial

The flat overlay for liquid pools — water, oil, poison, blood — plus the electrified and frozen variants. Built for a single-segment ground plane.

```ts
import { buildPoolSurfaceMaterial } from '@artificer-forge/vfx'

// tex: R=water G=oil B=poison A=blood
// stateTex: R=electrified charge, G=frozen
// electricTex (optional): greyscale lightning-bolt sprite for arc effects
const mat = buildPoolSurfaceMaterial(tex, stateTex, electricTex)
plane.material = mat
```

- Pool edges are broken up by slow-drifting fractal noise so they read organic, not disc-shaped.
- Oil gets a faked cosine-rainbow thin-film sheen; water is low-opacity so the ground shows through.
- Frozen pools become a cracked-plate Worley ice sheet with a glassy specular drop; electrified pools flash randomly-angled bolt sprites with a cyan (or red, over blood) halo.

### buildFireSurfaceMaterial

The additive flame layer for a fire surface. Takes the fire `DataTexture` (R = per-cell fire amount).

```ts
import { buildFireSurfaceMaterial } from '@artificer-forge/vfx'

const mat = buildFireSurfaceMaterial(fireTex)
firePlane.material = mat
```

Two-pass world-anchored noise (low-freq lateral warp + high-freq flame body) shaped into tongues that taper as they rise, with a 4-stop heat ramp (ember-red → orange → yellow → white core) and an in-place breathing pulse. `AdditiveBlending`, transparent, no depth write.

### buildCharcoalSurfaceMaterial

The glowing coal bed beneath a fire. Unlike the flat pool/fire planes, this expects a **subdivided plane** so it can actually displace geometry vertically — the lumps have real silhouette and catch light.

```ts
import { buildCharcoalSurfaceMaterial } from '@artificer-forge/vfx'

const mat = buildCharcoalSurfaceMaterial(fireTex)
charcoalPlane.material = mat   // mesh: subdivided PlaneGeometry, rotateX(-PI/2), origin at (0,0)
```

It displaces humps up by `hump × fire coverage`, finite-differences the height field to recompute normals (Three.js doesn't do this after a vertex displacement), and pools an ember glow in the valleys between humps with a spatially-offset flicker. `depthWrite` stays **on** here (with a tiny `alphaTest`) so it sorts correctly against the additive fire billboards.

::alert{type="info"}
`buildCharcoalSurfaceMaterial` has a strict mesh contract: a full-field `PlaneGeometry`, `rotateX(-Math.PI / 2)`, positioned at x=0, z=0. The XZ-at-origin placement is what lets the vertex stage (`positionLocal.xz`) and fragment stage (`positionWorld.xz`) share the same noise field.
::
