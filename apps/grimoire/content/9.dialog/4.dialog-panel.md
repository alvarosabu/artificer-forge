title: Dialog Panel
description: The DialogPanel HUD component — renders the active line, speaker portrait, choices, and skill-check feedback.
---

`DialogPanel` is the 2D overlay that renders an active dialog: the speaker's portrait and name, the current line, and the clickable choices. It's a pure consumer of `useDialogStore` — it reads state and calls `choose` / `close`, holding no dialog logic of its own.

```ts
import { DialogPanel } from '@artificer-forge/engine/ui'
```

## Usage

Mount it in the HUD layer (the DOM overlay, not the canvas). It teleports itself to `<body>` and pins to the bottom-center of the screen, so placement in your template is just about where it lives in the component tree, not on-screen position.

```vue
<script setup lang="ts">
import { DialogPanel } from '@artificer-forge/engine/ui'
</script>

<template>
  <!-- in the HUD / overlay layer -->
  <DialogPanel />
</template>
```

It takes no props — everything comes from the store. The panel only shows while `dialogStore.isOpen` is true, animating in and out with a slide/fade transition.

## What it renders

- **Portrait** — a live-baked 3D portrait of the current speaker via `usePortraitRenderer` (the same source the action bar and character panel use), falling back to the speaker's first initial when there's no model (e.g. a Narrator).
- **Speaker name** — `currentSpeakerEntity?.name`, defaulting to `Narrator`.
- **Line** — `dialogStore.currentText`.
- **Choices** — one button per `currentChoices` entry. Each shows its number, an optional `[tagPrefix]` label, a `[SKILL DC]` badge for check choices, the choice text, and — for locked choices — the `lockedReason`. Unavailable choices render dimmed and disabled.
- **Check flash** — when a skill check resolves, `lastCheck` drives a brief badge showing the skill, DC, the roll math (`roll±modifier = total`), and SUCCESS / FAILURE.

## Input

The panel binds keyboard shortcuts while a dialog is open:

| Key | Action |
|-----|--------|
| `1`–`9` | Pick the matching choice (if available) |
| `Escape` | Close the dialog |

Clicking an available choice calls `dialogStore.choose(choice.index)` — note it passes the choice's `index`, which is stable even when locked-and-hidden choices are filtered out of the visible list.

::alert{type="info"}
`DialogPanel` is part of `@artificer-forge/engine/ui`, the Nuxt UI based HUD layer. It uses `UIcon` for the check-result badge, so the consuming app must have `@nuxt/ui` installed (an optional peer of the engine).
::
