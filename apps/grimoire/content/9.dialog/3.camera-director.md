---
title: Camera Director
description: useDialogCamera and the DialogCameraDirector component — cinematic framing that reacts to dialog state.
---

The camera director gives dialog its cinematic feel. It watches the dialog store and, every time the current node changes, tweens the active camera to frame the speaker according to the node's `cameraShot`. When the dialog ends it restores the camera to wherever it was before the conversation started.

```ts
import { DialogCameraDirector, useDialogCamera } from '@artificer-forge/engine/runtime'
```

## DialogCameraDirector

`DialogCameraDirector` is the component you drop in. It's an invisible `<TresGroup>` whose only job is to run `useDialogCamera()` from inside the canvas — the composable needs `useTresContext()` to reach the active camera, so it must live under `<TresCanvas>`.

```vue
<template>
  <!-- inside the TresCanvas / Game default slot -->
  <Character v-for="e in partyCharacters" :key="e.id" :entity-id="e.id" />
  <Actor v-for="e in actorEntities" :key="e.id" :entity-id="e.id" />

  <DialogCameraDirector />
  <Floor />
</template>
```

::alert{type="warning"}
`DialogCameraDirector` (and `useDialogCamera`) must be mounted **inside** the `<TresCanvas>` context. Mounting it in the HUD or plain DOM means `useTresContext()` can't resolve the active camera and the framing never runs.
::

## useDialogCamera

`useDialogCamera()` is the logic. Call it directly only if you're building a custom in-canvas wrapper; otherwise use the component. It:

- Subscribes to `dialogStore.currentNodeId`. On a change to a new node it frames the new speaker; when it clears, it restores.
- Takes a one-time snapshot of the camera's pre-dialog transform and restores it on close.
- Drives all motion with a GSAP tween (`power2.inOut`), so rapid node changes interpolate from the in-flight transform rather than snapping.
- Cleans up its tween on unmount.

It returns nothing — it's a side-effecting subscription, not an API surface.

## Shot presets

Each node picks its framing via `cameraShot` (default `three-quarter`). The director computes a camera position and look-at target from the speaker's world position, the listener (the party leader), and the chosen preset:

| `cameraShot` | Framing |
|--------------|---------|
| `three-quarter` | Default. Anchored on the speaker, offset to one side at a 3/4 angle — both characters in frame |
| `over-shoulder` | Behind and above the listener, looking past their shoulder at the speaker |
| `closeup` | Pulled in tight on the speaker, slightly off-axis |
| `wide` | Pulled-back reaction shot |
| `two-shot` | Both speakers framed from a point perpendicular to the line between them |

The "axis" is the ground-plane direction from listener to speaker; presets offset along and perpendicular to it. With no listener it falls back to framing straight off the speaker. `two-shot` instead sits perpendicular to the midpoint and pulls back proportional to how far apart the two characters stand.
