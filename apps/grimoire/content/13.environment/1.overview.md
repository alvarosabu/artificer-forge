The environment system provides procedural tools for populating 3D scenes with vegetation, terrain, and atmospheric elements. All components are designed for WebGPU rendering via TresJS and Three.js's TSL (Three.js Shading Language).

## Available Systems

| System | Import | Description |
|--------|--------|-------------|
| [Foliage](/environment/foliage) | `@artificer-forge/engine/runtime` | Procedural billboard sphere clusters for trees and bushes |

## Architecture

Environment components follow the **reference-driven instancing** pattern:

1. Define placement points as `Object3D` references (position + scale)
2. Pass references to the environment component
3. The component generates GPU-instanced geometry from those references
4. A single draw call renders all instances

This keeps the scene graph simple while leveraging GPU instancing for performance.

```vue
<template>
  <TresCanvas>
    <Foliage
      :references="treePlacements"
      :color-a="'#b4b536'"
      :color-b="'#d8cf3b'"
    />
  </TresCanvas>
</template>
```

::alert{type="info"}
All environment shaders use TSL (`three/tsl`) and WebGPU node materials (`three/webgpu`). Your renderer must be `WebGPURenderer`.
::
