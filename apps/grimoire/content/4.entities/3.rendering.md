---
title: Rendering Entities
description: Render store entities in 3D scenes with smart/dumb component pattern
---

The experience page (smart) manages entity lifecycle and passes entity IDs to the engine's `Character` and `Interactable` components (dumb) for rendering. Those components are exported from the runtime layer:

```ts
import { Character, Floor, Interactable } from '@artificer-forge/engine/runtime'
```

::note
The engine's in-scene components are **entity-id driven** — you pass `:entity-id`, not raw `model`/`position` props. Each component reads its own state from `useGameStore`, so the scene stays in sync with the store automatically.
::

## Pattern: Smart vs Dumb

| Component | Responsibilities |
|-----------|-----------------|
| **Experience** (smart) | Access store, spawn entities, compute filtered lists |
| **Character** (dumb) | Receive props, load model, play animations |

This keeps Character reusable and testable - it doesn't need store access.

## Experience Page

```vue
<!-- pages/model/experience.vue -->
<script setup lang="ts">
import { Character, Floor, useGameStore } from '@artificer-forge/engine/runtime'

const gameStore = useGameStore()

// Track spawned entity for cleanup
const heroEntityId = ref<string | null>(null)

onMounted(async () => {
  // Spawn hero at origin
  const entityId = await gameStore.spawnFromTemplate('hero', { x: 0, y: 0, z: 0 })
  heroEntityId.value = entityId
  gameStore.addToParty(entityId)
})

// Filter entities by type for rendering
const characterEntities = computed(() => {
  return [...gameStore.entities.values()].filter(e => e.type === 'character')
})
</script>

<template>
  <TresFog :args="['#020420', 10, 30]" />
  <TresPerspectiveCamera :position="[5, 5, 5]" :near="0.1" :far="100" />
  <OrbitControls />

  <TresAmbientLight :intensity="0.8" />
  <TresDirectionalLight
    :position="[5, 5, 5]"
    :intensity="1.5"
    cast-shadow
  />

  <!-- Render character entities from store -->
  <Suspense>
    <Character
      v-for="entity in characterEntities"
      :key="entity.id"
      :entity-id="entity.id"
    />
  </Suspense>

  <TresAxesHelper />
  <Floor />
</template>
```

## The Character Component (Dumb)

The engine's `Character` is the dumb renderer. It takes a single `:entity-id` prop and reads everything else — model path, rig, position, animations, equipment, status effects — from `useGameStore`. It handles GLTF loading, the rig, animation playback, equipment bone attachment, and damage/status overlays internally.

```vue
<script setup lang="ts">
import { Character } from '@artificer-forge/engine/runtime'
</script>

<template>
  <Suspense>
    <Character :entity-id="entity.id" />
  </Suspense>
</template>
```

It loads the model with `useGLTF`, resolves the rig from the entity's `rig` field (default `Rig_Medium`), and drives animations via [`useCharacterAnimations`](/characters/animations) — all keyed off the store, so you never pass raw `model`/`position` props.

## Spawning Multiple Entities

```ts
onMounted(async () => {
  // Spawn party
  const heroId = await gameStore.spawnFromTemplate('hero', { x: 0, y: 0, z: 0 })
  const playerId = await gameStore.spawnFromTemplate('player', { x: -1, y: 0, z: 1 })

  gameStore.addToParty(heroId)
  gameStore.addToParty(playerId)

  // Spawn enemies
  await gameStore.spawnFromTemplate('goblin_scout', { x: 5, y: 0, z: 0 })
  await gameStore.spawnFromTemplate('goblin_scout', { x: 6, y: 0, z: 2 })
  await gameStore.spawnFromTemplate('goblin_warrior', { x: 5.5, y: 0, z: 1 })
})
```

## Filtered Rendering

Render different entity types with different components:

```vue
<script setup>
const characterEntities = computed(() =>
  [...gameStore.entities.values()].filter(e => e.type === 'character')
)

const itemEntities = computed(() =>
  [...gameStore.entities.values()].filter(e => e.type === 'item')
)

const interactableEntities = computed(() =>
  [...gameStore.entities.values()].filter(e => e.type === 'interactable')
)
</script>

<template>
  <!-- Characters with animation support -->
  <Suspense>
    <Character
      v-for="entity in characterEntities"
      :key="entity.id"
      :entity-id="entity.id"
    />
  </Suspense>

  <!-- Items (world drops) -->
  <Suspense>
    <WorldItem
      v-for="entity in itemEntities"
      :key="entity.id"
      :entity-id="entity.id"
    />
  </Suspense>

  <!-- Interactables (chests, doors) -->
  <Suspense>
    <Interactable
      v-for="entity in interactableEntities"
      :key="entity.id"
      :entity-id="entity.id"
    />
  </Suspense>
</template>
```

`Character`, `Interactable`, and `Floor` come from `@artificer-forge/engine/runtime`; `WorldItem` (the in-world item drop) comes from `@artificer-forge/engine/ui`.

## Reactive Updates

When store state changes, the scene updates automatically:

```ts
// Move an entity - scene re-renders
gameStore.updateEntity(entityId, {
  position: { x: newX, y: 0, z: newZ }
})

// Remove entity - disappears from scene
gameStore.removeEntity(entityId)

// Update HP - could trigger death animation
const entity = gameStore.getEntity(entityId)
if (entity && entity.hp !== undefined) {
  const newHp = entity.hp - damage
  gameStore.updateEntity(entityId, { hp: newHp })

  if (newHp <= 0) {
    // Trigger death, remove after animation
    characterRef.value?.play('death')
    setTimeout(() => gameStore.removeEntity(entityId), 2000)
  }
}
```

## Entity References for Animation Control

To control individual character animations, register each component with `useSceneRefs` (see [Scene Refs](/core-concepts/scene-refs)):

```vue
<script setup>
import { Character, useSceneRefs } from '@artificer-forge/engine/runtime'

const { setCharacterRef, getCharacterRef } = useSceneRefs()

function playAnimation(entityId: string, anim: string) {
  getCharacterRef(entityId)?.play(anim)
}
</script>

<template>
  <Suspense>
    <Character
      v-for="entity in characterEntities"
      :key="entity.id"
      :ref="(el) => setCharacterRef(entity.id, el)"
      :entity-id="entity.id"
    />
  </Suspense>
</template>
```

::note
Wrap async model loading components in `<Suspense>` to handle loading states gracefully.
::
