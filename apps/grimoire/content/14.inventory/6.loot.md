Loot is the world-side view of the inventory system. The same `moveItem` primitive handles "take this potion from a chest" and "swap my sword in the inventory modal" — only the source container differs.

The loot popover state composable lives in the runtime layer; its widgets ship from the UI layer:

```ts
import { useLoot } from '@artificer-forge/engine/runtime'
import { LootPopover, WorldItem } from '@artificer-forge/engine/ui'
```

## Three Kinds of Lootable Source

| Source | How items get there | Lifecycle |
|--------|---------------------|-----------|
| Chest | `lootTable` rolled in `spawnFromTemplate` | Persists until emptied |
| Corpse | Character's existing items + equipment | Created when `hp <= 0` |
| World drop | `dropItem(id, position)` or scene-placed `WorldItem` | Persists until picked up |

All three are entities — the only difference is the value of `containerId` on their contained items (and `null` for world drops).

## Loot Tables

Chests and other lootable interactables declare a `lootTable` on the template:

```yaml
# content/entities/interactables/chest-common.yaml
templateId: chest_common
type: interactable
subtype: container
name: Wooden Chest
lootTable:
  - id: healing_potion
    quantity: 1
    chance: 1.0
  - id: shortsword
    quantity: [1, 2]
    chance: 0.4
```

When `spawnFromTemplate` instantiates a chest, it walks the table:

1. Roll `Math.random()` against `chance`.
2. If `quantity` is a number, use it. If it's `[min, max]`, pick uniformly.
3. Call `spawnItemEntity(id, { containerId: chestInstanceId, quantity })`.

The chest ends up with item entities reparented to it. `itemsIn(chestId)` returns them.

::note
Scenes can override `lootTable` per-instance: a "boss chest" placed in a scene can specify its own table without a new template.
::

## Corpses as Containers

When a character dies (`hp <= 0` or `dead === true`), they become a lootable container. No special data structure — their existing items just become accessible because the loot UI checks for it.

Key bits in `LootPopover`:

```ts
const isDeadContainer = computed(() =>
  container.value?.type === 'character'
  && ((container.value.hp ?? 1) <= 0 || !!container.value.dead),
)
const items = computed(() =>
  gameStore.itemsIn(loot.state.containerId, { includeEquipped: isDeadContainer.value }),
)
```

Two things are notable:

1. `includeEquipped: true` so the dead character's *equipped* gear shows up in the loot grid. Living characters' equipment never appears in their own bag view.
2. The doll in the world still wears the equipped items (the `useEquipment` bone attach doesn't care that the owner is dead). Picking up an equipped item from a corpse strips it from the bone.

The [item context menu](/inventory/interactions#actions-by-owner) only offers **Pick Up** on dead-owner items — never Unequip or Drop.

## LootPopover

A small popover anchored to the spot the player right-clicked.

```vue
<InventoryLootPopover />
```

Triggered via `useLoot`:

```ts
const loot = useLoot()
loot.open(containerId, screenX, screenY)
```

What it shows:

- Container name + close button
- A 6-column grid of `ItemCell`s — each right-clickable to bring up the context menu (Pick Up, Transfer to…)
- **Take All** button at the bottom

### Take All & Overflow

`takeAll` walks the grid and moves every item to the party leader. Any failures (most commonly `over-capacity`) are collected and surfaced via Nuxt UI's toast:

```ts
const failed: string[] = []
for (const item of items.value) {
  const result = gameStore.moveItem(item.id, { containerId: leader })
  if (!result.ok) failed.push(item.name)
}
if (failed.length) {
  toast.add({
    title: 'Could not take everything',
    description: `${failed.length} item(s) left behind — inventory full or invalid.`,
    color: 'warning',
  })
}
```

The popover auto-closes once the container is empty.

## World Drops & WorldItem

Items with `containerId === null` exist in the world. They're rendered by the `WorldItem` component, which loads the item's GLB model and places it at `item.position`:

```vue
<Suspense
  v-for="worldItem in worldItems"
  :key="worldItem.id"
>
  <InventoryWorldItem :item="worldItem" />
</Suspense>
```

```ts
const worldItems = computed(() =>
  [...gameStore.entities.values()].filter(
    e => e.type === 'item' && (e.containerId === null || e.containerId === undefined) && !!e.model,
  ),
)
```

### Right-click to Pick Up

`WorldItem` forwards Three.js `contextmenu` events to the global `useContextMenu`, which the scene page handles:

```ts
onAction((action, entityId) => {
  if (action === 'pickup') {
    const recipient = gameStore.selectedEntityId ?? gameStore.party.leader
    if (recipient) gameStore.pickupItem(entityId, recipient)
  }
})
```

This is the same dispatcher used for "examine" and "loot" — the [Entity context menu](/entities/interactables) page covers it in detail.

## Walk-To-Loot Pattern

For chests and corpses, players don't just teleport — the leader walks to the container before the popover opens. The corpse scene page wires this up:

```ts
function walkToAndLoot(entityId: string) {
  const target = gameStore.getEntity(entityId)
  const player = gameStore.selectedEntity
  const characterRef = getCharacterRef(player.id)

  const direction = playerPos.clone().sub(targetPos).normalize()
  const approach = targetPos.clone().add(direction.multiplyScalar(INTERACTION_DISTANCE))

  const { off } = characterRef.onArrive(() => {
    loot.open(entityId, screenX, screenY)
    off()
  })

  characterRef.moveTo(approach)
}
```

The `onArrive` callback fires once the character finishes its move-to path; the loot popover opens at that point.

## Drop From Inventory

The reverse of pickup. `dropItem(id, position)` reparents the item to `null` at a position — typically the dropper's current position:

```ts
gameStore.dropItem(itemId, owner.position)
```

`WorldItem` picks it up on the next render and the item is now visible on the ground, right where it was dropped.

## Recap

The loot subsystem is mostly UI + a handful of conventions:

- **Items as entities** means corpses and chests share one code path with bags.
- **`includeEquipped: true`** is the one flag that turns equipped gear into loot.
- **`Take All`** is just a loop over `moveItem` that surfaces failures as a toast.
- **World items** are entities with `containerId === null`; their position determines where they render.
